# 대학 메인 화면 UI 실행 기획안 (상업용 품질 목표)

## 0) 사용 스킬 고지
- 이번 요청은 `skill-creator`, `skill-installer` 범주(스킬 생성/설치)에 해당하지 않아 별도 스킬을 사용하지 않음.

## 1) 목표 및 범위
### 목표
- 현재 대학 화면(팀 순위/리더보드/빅보드/스카우팅)을 **상업용 게임 UI 수준**으로 고도화한다.
- 사용자가 첫 3초 내에 "정보가 잘 정리되어 있고 프로덕트 완성도가 높다"고 느끼는 수준의 시각/정보/인터랙션 품질을 확보한다.

### 범위 (엄수)
- **수정 가능 파일:** `static/NBA.html`, `static/NBA.css`, `static/NBA.js`
- **수정 금지:** 백엔드 API, 파이썬 로직, DB 스키마, 타 화면(메인/전술/일정 등) 구조 변경
- **데이터 제약:** 기존 조회 API만 활용 (신규 API 생성 금지)

### 성공 기준 (Definition of Done)
1. 4개 탭(팀 순위/리더보드/빅보드/스카우팅)이 공통 디자인 시스템으로 통일됨
2. 각 탭이 `loading/empty/error/normal` 상태를 명확히 표현함
3. 테이블 중심 UI가 "분석 대시보드" 느낌으로 시각화(배지/칩/미니바/하이라이트)됨
4. 스카우팅 탭이 폼 화면이 아닌 "의사결정 화면"으로 전환됨
5. 기존 API 호출 경로와 파라미터는 유지됨

---

## 2) 현재 구현 자산(재사용 기준)
아래 API/DOM 자산은 이미 존재하므로 이를 중심으로 재구성한다.

### 기존 API 호출(유지)
- 메타/팀: `/api/college/meta`, `/api/college/teams`
- 팀 상세: `/api/college/team-detail/{teamId}`
- 리더보드: `/api/college/players?sort=...&order=desc&limit=...`
- 빅보드: `/api/offseason/draft/bigboard/expert?expert_id=...&pool_mode=auto`
- 스카우팅: `/api/scouting/scouts/{teamId}`, `/api/scouting/reports?team_id=...&limit=...`
- 스카우팅 액션: `/api/scouting/assign`, `/api/scouting/unassign`

### 기존 DOM 앵커(유지/확장)
- 탭/패널: `college-tab-*`, `college-panel-*`
- 테이블 바디: `college-teams-body`, `college-leaders-body`, `college-bigboard-body`, `college-reports-body`
- 컨트롤: `college-leader-sort`, `college-expert-select`, `college-scout-select`, `college-scout-player-select`

> 핵심 원칙: **ID/이벤트 핸들러는 가급적 유지**하고, 레이아웃 래퍼/보조 컴포넌트를 추가하는 방식으로 리스크 최소화.

- "Broadcast + Front Office"
- TV 중계 그래픽의 임팩트 + GM 데이터 대시보드의 신뢰감

- `--text-main: #E7ECF5`

- `--accent: #4DA3FF`

- Spacing: 8pt 시스템 (`8, 12, 16, 24, 32`)

## 타이포 가이드
- 제목: 30/36, 700
- 섹션 헤드: 22/28, 700

## 4) 정보 구조(IA) 재설계
## 상단 공통 헤더
1. 좌측: `COLLEGE LEAGUE` + `대학 리그`
2. 중앙: KPI 배지 3개
   - 시즌
   - 대학팀 수
   - 예정 드래프트 연도
3. 우측: 액션(`메인으로`) + 현재 탭 컨텍스트

## 탭 구조
- 기존 4탭 유지
- 탭 버튼을 segmented control 스타일로 변경
- 활성 탭은 배경+하단 인디케이터 동시 제공

## 공통 레이아웃
- 데스크톱 기준 12컬럼
- 메인 컨텐츠는 2단 기반(8:4 또는 7:5)
- 테이블 카드와 상세 카드의 시각 위계를 분리

---

## 5) 탭별 화면 상세 기획
## A. 팀 순위(Teams)
### 목표
- 리그 전체 파워 밸런스와 선택 팀 로스터를 한눈에 이해

### 좌측 카드: 대학 팀 순위
- 컬럼: RK / 팀 / 컨퍼런스 / W / L / SRS (기존 유지)
- 개선 요소
  1. RK 앞 순위 상태 아이콘(기본 `—`)
  2. SRS 미니바(클라이언트 min-max 정규화)
  3. 상위 4팀 하이라이트, 선택 행 active 처리
  4. 팀명 좌측 이니셜 배지

### 우측 카드: 선택 팀 로스터
- 제목: `{팀명} 로스터`
- 컬럼 유지 + 개선
  - POS 배지형
  - 학년 숫자에 `FR/SO/JR/SR` 보조 라벨(학년 숫자 매핑)
  - PTS/REB/AST 우측 정렬 + 1자리 고정

### 상태 설계
- 로딩: 스켈레톤 10행
- Empty: "대학 팀 데이터를 불러오지 못했습니다"
- Error: 재시도 버튼 포함 인라인 경고

---

## B. 리더보드(Leaders)
### 목표
- 시즌 진행 전/후 모두 의미 있는 화면 제공

### 헤더 컨트롤
- 정렬 드롭다운 + 빠른 프리셋 칩(PTS/REB/AST/STL/BLK)
- 현재 정렬 컬럼 헤더 강조

### 테이블 개선
- Top3 메달 마커(🥇/🥈/🥉 또는 컬러 도트)
- 스탯 셀 heat tint (현재 목록 내 상대값)
- 팀명 컬럼 폭 확장 + 말줄임 + tooltip

### 0.0 데이터 대응
- 전체 값이 0이면 상단 안내 배너
  - "시즌 개막 전 데이터입니다. 시뮬레이션 진행 후 통계가 갱신됩니다."

---

## C. 빅보드(Big Board)
### 목표
- "선수 목록"이 아니라 "드래프트 의사결정" 화면으로 전환

### 레이아웃
- 좌: 전문가 빅보드 테이블
- 우: 선택 선수 Prospect 카드(신규 UI 컴포넌트, 기존 데이터 기반)

### 빅보드 테이블 개선
- Tier별 색상칩 (`Tier 1`, `Lottery`, `Late 1st`...)
- 요약 문자열 파싱:
  - `Strengths:` 이후 키워드 칩
  - `Concern:` 이후 위험 칩

### 전문가 선택 UX
- 기존 select 유지 + 상단 persona 라벨 표시
- 전문가 변경 시 120~180ms 페이드 전환

---

## D. 스카우팅(Scouting)
### 목표
- 배정/해제를 "작업 버튼"이 아닌 "인사이트 생산 흐름"으로 제시

### 좌측 카드: 스카우터 배정
- `스카우터`, `대상 선수` 필드 유지
- 스카우터 선택 시 보조 정보 표시
  - 전문 분야, 현재 배정 여부(가능하면 reports 기준 추정)
- CTA 우선순위
  - Primary: 배정
  - Secondary: 해제

### 우측 카드: 스카우팅 리포트
- 테이블 유지하되 상태칩 도입
- 빈 상태를 타임라인형 메시지로 교체
  - "배정 후 월말 진행 시 리포트가 생성됩니다"
  - 바로 아래 최근 배정 가이드를 보조 텍스트로 제공

---

## 6) 인터랙션/모션 규칙
- Hover: 120ms ease-out
- Panel/tab 전환: 180ms
- 카드 등장: 220ms translateY + fade
- 버튼/입력 포커스 링: `2px solid var(--accent)`
- 모든 인터랙션은 `prefers-reduced-motion` 대응

---

## 7) 컴포넌트 규격 (구현용)
## 공통 카드
- 패딩 20~24px
- 보더 1px `var(--line)`
- 라운드 16px

## 테이블
- header height 50px
- row height 46px
- zebra는 사용하지 않고 hover/selected로만 강조
- 수치 컬럼 오른쪽 정렬

## 칩/배지
- height 24px
- horizontal padding 10px
- font 12/16 semibold

## 빈 상태(Empty)
- 아이콘 + 1줄 제목 + 1줄 설명 + 선택적 액션 버튼
- 최소 높이 180px 보장

---

## 8) 접근성/완성도 체크리스트
1. 텍스트 대비 4.5:1 이상
2. 키보드 탭 순서가 시각 흐름과 일치
3. `aria-selected`, `aria-controls`, `aria-live` 적절 반영
4. 드롭다운/버튼 hit area 40px 이상
5. 한글 용어 일관성(예: 리더보드/스카우팅 표기 통일)

---

## 9) 구현 단계(다음 작업용)
## Phase 1: 구조 정리 (HTML)
- 대학 화면 섹션에 공통 헤더/KPI/컨텐츠 래퍼 추가
- 기존 ID는 유지하고 래퍼 클래스만 증설

## Phase 2: 디자인 시스템 반영 (CSS)
- 대학 화면 전용 네임스페이스(예: `.college-*`)로 토큰/카드/테이블/칩 스타일 적용
- 다른 화면 영향 없는 선택자 범위로 제한

## Phase 3: 데이터 표현 강화 (JS)
- 기존 fetch 로직 유지
- 렌더 함수에 부가 표현 추가
  - 미니바/heat tint/칩 파싱/empty-state 분기

## Phase 4: QA
- 탭 4개 전환, 정렬, 전문가 변경, 배정/해제 회귀 테스트
- 데이터 없음/오류 응답/초기 로딩 상태 점검

---

## 10) 비기능 요구사항
- 성능: 탭 전환 체감 200ms 내 반응
- 안정성: API 실패 시 전체 화면 붕괴 금지(부분 오류 처리)
- 유지보수성: 대학 화면 스타일/스크립트를 기존 구조에서 독립적으로 추적 가능

---

## 11) 최종 산출물 요구사항 (다음 구현 작업의 완료 조건)
1. 대학 화면 전용 UI 갱신이 `NBA.html/css/js`에만 반영됨
2. 기존 API 호출 시그니처/경로 변경 없음
3. 스크린샷 기준으로 4탭 모두 동일 디자인 언어를 공유함
4. 빈 상태/로딩/오류 상태가 시각적으로 완성됨
5. "바이브 코딩 산출물" 느낌이 아닌 정돈된 제품 UI 품질 달성
