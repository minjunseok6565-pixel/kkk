# 트레이드 이슈 2 구현 검증 리포트 (현 브랜치)

작성일: 2026-03-10  
검증 대상 이슈: **(2) 단일 archetype·salary×years → 역할 질감 + 계약 질감**

---

## 1) 검증 결론 (요약)

현 브랜치 기준으로 이슈 2는 **"신규 모듈 구현"은 완료됐지만, "실제 의사결정 경로 반영"은 미완료** 상태다.

- ✅ 완료된 부분
  - `role_texture.py`에서 다차원 역할 질감(`creation/spacing/rim_pressure/defense + connector`) 산출 로직이 구현됨.
  - `contract_texture.py`에서 계약 질감(`guaranteed_commitment/control_direction/reversibility/trigger_risk/matching_utility/toxic_risk`) 산출 로직이 구현됨.
  - `context_v2.py`가 두 모듈을 조립해 `ValuationContextV2`를 생성할 수 있음.

- ⚠️ 미완료(핵심)
  - 실제 딜 평가의 package 경로(`package_effects.py`)는 여전히
    - 단일 archetype 버킷(`_primary_archetype_tag`) 기반 중복 페널티,
    - `salary * years` 근사(`_commitment_metric`) 기반 CAP_FLEX 보정
    을 사용 중.
  - 즉, **신규 role/contract texture가 기존 package 의사결정에 아직 연결되지 않음**.

## 2) 코드 기준 상세 점검

## 2-1. 역할 질감(Role Texture) 모듈 상태

`trades/valuation/role_texture.py`

- `RoleTexture` 데이터 구조가 설계 문서와 부합하는 핵심 필드를 제공.
- 입력 우선순위(`meta.role_fit` → `attrs.role_fit`), 누락 시 coverage/notes 플래그 처리 구현.
- `FitEngine` supply 벡터를 보강 입력으로 사용해 축 점수를 상향 보정.

판정: **모듈 자체 구현은 적절**.

## 2-2. 계약 질감(Contract Texture) 모듈 상태

`trades/valuation/contract_texture.py`

- `ContractTexture` 핵심 필드 구현.
- `contracts/terms.py`의 `remaining_years`, `remaining_salary_total`, `salary_schedule`, `salary_for_season`를 사용하여 SSOT 재사용 원칙 준수.
- 옵션 방향(TO/PO/ETO)에 따른 `control_direction` 반영.

판정: **모듈 자체 구현은 적절**.

## 2-3. v2 오케스트레이션 연결 상태

`trades/valuation/context_v2.py`

- `build_valuation_context_v2()`에서 `build_role_textures()`, `build_contract_textures()`를 호출하고 결과를 `ValuationContextV2`에 담음.
- 하지만 repo 전체 검색 기준, `build_valuation_context_v2()`는 아직 실제 valuation 서비스 주 경로에서 호출되지 않음.

판정: **v2 컨텍스트 생성 가능하나, 운영 경로 미연결**.

## 2-4. 레거시 경로 잔존 상태 (중요)

`trades/valuation/package_effects.py`

- `_primary_archetype_tag()`가 여전히 단일 태그로 incoming player를 버킷화하여 diminishing returns 계산.
- `_commitment_metric()`가 여전히 `salary_now * remaining_years` 근사치를 계산.
- `_cap_flex_adjustment()`에서 commitment delta 계산 시 `_commitment_metric()`을 직접 합산.

판정: **이슈 2의 핵심 레거시 경로가 활성 상태로 남아 있음**.

## 3) "현재 브랜치 기준" 추가 필요 작업

우선순위 순으로 정리:

1. **패키지 평가 경로에 v2 texture 주입**
   - `package_effects`에 `basketball_component`(role_texture 기반) / `contract_component`(contract_texture 기반) 분리 계산 경로 추가.
   - 최소 dual-read 기간 동안 v1/v2 동시 계산 후 diff telemetry 축적.

2. **단일 archetype 버킷 의존 제거 준비**
   - diminishing returns 입력을 `RoleTexture` 다차원 score matrix로 대체.
   - 기존 `tag -> list[player]` 버킷 구조를 `texture similarity / overlap` 기반 구조로 교체.

3. **commitment 선형 근사 경로 제거 준비**
   - CAP_FLEX 관련 future commitment를 `ContractTexture.guaranteed_commitment + control_direction/trigger_risk` 결합 점수로 교체.
   - `_cap_flex_adjustment()` 내부 commitment 합산식을 texture 기반으로 변경.

4. **service 주 경로 연결**
   - `valuation/service.py`(또는 evaluator adapter)에서 `context_v2`를 feature flag로 호출 가능하게 연결.
   - flag off 시 기존 결과 100% 동일, on 시 v2 점진 반영.

5. **회귀/검증 테스트 추가**
   - role/contract texture 단위 테스트.
   - package_effects v1 vs v2 diff 스냅샷 테스트.
   - 옵션 구조가 다른 동일 연봉 계약 케이스에서 반응 차이 검증 테스트.

## 4) 삭제 대상 레거시 코드 (조건부)

아래 항목은 **v2 반영 경로 안정화 이후** 삭제 권장:

- `package_effects.py::_primary_archetype_tag`
- `package_effects.py::_commitment_metric`
- `_diminishing_returns()` 내부 단일 archetype 버킷 로직
- `_cap_flex_adjustment()`의 `_commitment_metric` 직접 합산 경로

삭제 전제 조건:

1. feature flag on/off 회귀 통과
2. dual-read 1 릴리즈 이상 diff telemetry 확보
3. 딜 다양성/설명가능성 지표 이상 없음 확인

으로 판정한다.
