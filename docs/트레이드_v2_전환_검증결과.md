## 1) 검증 범위

요청 범위에 맞춰 아래 4개 이슈의 **현재 브랜치 구현 상태**를 점검했다.

## 2) 결론 요약

**판정: `부분 전환(Partial Cutover)` 상태**

- 핵심 5모듈(`context_v2`, `pick_distribution`, `role_texture`, `contract_texture`, `cap_ledger`)은 생성되어 있고, 실제 평가 경로에 일부 연결됨.
- 이슈 1/2/7/8의 핵심 방향성은 대체로 반영됨.
- 다만, v2 완전 전환이라고 보기엔 다음이 남아 있음:
  1. v1 호환 경로(legacy fallback) 다수 잔존
  2. 일부 경로에서 v2 산출을 재사용하지 않고 재계산(중복 책임)
  3. pick distribution 입력 타입 일관성 결함(테스트 실패)

## 3) 이슈별 검증 결과

## 이슈 1) 픽 기대값 → 분포/시나리오 자산

### 확인된 개선
- `PickDistributionBundle` 기반 PMF/CDF/EV/분산/tail 값을 생성하는 경로가 구현됨.
- 보호(`resolve_pick_protection`)와 스왑(`resolve_swap_outcome`) 의미론을 분포 구성 단계에서 반영함.
- 서비스 레이어에서 `context_v2`를 사용하면 pick expectation은 distribution의 `compat_expected_pick_number`로 주입됨.
- 마켓 프라이싱에서 분포 기반 가산(`PICK_DISTRIBUTION_*`)을 반영하고, legacy 보호/스왑 근사 보정 스텝은 스킵되는 방향으로 구성됨.

### 미해결/리스크
- `MarketPricer._price_pick()`이 distribution을 객체처럼 직접 참조(`distribution.compat_expected_pick_number`)하는 코드가 있어, dict 주입 테스트에서 실패함.
- `data_context.py`의 `build_pick_expectations_from_standings()`는 여전히 유지되어 v1 경로가 공존함(의도된 dual-read 단계이지만 완전 전환은 아님).

### 판정
- **기능적 방향은 반영**, 하지만 **입력 타입 일관성 버그 + 레거시 경로 잔존**으로 완전 전환은 미완료.

---

## 이슈 2) 단일 archetype·salary×years → 역할 질감 + 계약 질감

### 확인된 개선
- `role_texture.py`에서 creation/spacing/rim_pressure/defense/connector 축을 계산.
- role_fit 소스 우선순위(`meta.role_fit` → `attrs.role_fit`) 및 missing coverage 기록 구현.
- `contract_texture.py`에서 계약 질감 지표(guaranteed/control/reversibility/trigger/matching/toxic) 계산.
- `package_effects.py`가 role texture 기반 overlap 계산 및 contract texture 기반 커미트먼트 가중을 사용.

### 미해결/리스크
- `package_effects.py`는 `context_v2`에서 이미 계산된 texture를 재사용하지 않고 내부에서 다시 `build_role_textures`, `build_contract_textures`를 호출함.
  - 즉, “단일 계산 책임(SSOT)” 관점에서는 아직 중복 경로.
- 구조상 `diminishing_factors` 기반 감쇠 로직은 유지되고 있어, 구상서의 “완전한 다차원 matrix 전환” 대비 일부는 과도기 형태.

### 판정
- **핵심 지표 전환은 완료에 가까움**, 단 **context_v2 중심 단일화는 미완료**.

---

## 이슈 7) CAP_FLEX 선형 근사 → 밸런스시트·옵셔널리티

### 확인된 개선
- `cap_ledger.py` 도입으로 `current_operating_room`, `future_locked_money`, `optionality_profile`, `liquidity_profile`, `apron_pressure`, `extension_pressure`, `posture_adjusted_flex` 계산 지원.
- `package_effects._cap_flex_adjustment()`는 payroll 정보가 있을 때 `build_cap_ledgers + score_cap_flex_delta` 경로를 우선 사용.

### 미해결/리스크
- payroll 불충분 시 `cap_flex_enable_legacy_commitment_fallback=True`로 계약 커미트먼트 기반 fallback이 동작.
- 즉, 선형 근사식이 완전히 제거된 상태는 아니며, “ledger 우선 + legacy 백업” 과도기.

### 판정
- **우선 경로는 v2로 전환됨**, 다만 **legacy fallback 제거 전 단계**.

---

## 이슈 8) 단일 버킷 중복 → 다차원 중복 + 로테이션 맥락

### 확인된 개선
- 중복 계산이 단일 archetype 태그가 아닌 RoleTexture 벡터 유사도(`_texture_overlap`)로 전환됨.
- 로테이션 맥락(`rotation_context`) 기반 완화(`_rotation_context_relief`)가 `ROSTER_EXCESS_WASTE`에 반영됨.
- hole penalty에 포지션 버킷 + texture axis parallel penalty가 함께 반영됨.

### 미해결/리스크
- depth 관련 일부 로직은 여전히 G/W/B 버킷 공급 계산이 병행됨(완전한 다차원 단일체계는 아님).
- 즉, “버킷 제거”가 아니라 “버킷 + texture 병렬” 구조.

### 판정
- **의도한 방향으로 크게 개선**, 다만 **버킷 로직의 잔존으로 완전 치환은 아님**.

## 4) v2 전환 상태 평가

### 현재 상태
- 서비스 엔트리포인트에서 `use_valuation_context_v2` 기본값이 `True`이며 stage(`shadow/gradual/full`)가 존재.
- `context_v2` diagnostics(diff report/coverage) 메타를 evaluation에 붙이는 telemetry 경로가 존재.

### 전환 성숙도 판단
- **전환 단계는 `dual-read + gradual rollout`에 부합**.
- **완전 cutover(레거시 삭제 완료)** 단계는 아님.

## 5) 추가로 필요한 작업 (우선순위)

1. **P0: pick distribution 입력 타입 정합성 수정**
   - `MarketPricer._price_pick()`에서 distribution을 dataclass/dict 양쪽 지원하도록 접근 통일.
   - 현재 테스트 실패 3건의 직접 원인.

2. **P1: context_v2 산출 재사용으로 중복 계산 제거**
   - `package_effects`에서 role/contract/cap 계산을 내부 재계산하지 말고 context_v2 결과 주입 방식으로 변경.
   - 계산 책임 단일화 + 성능 + 설명 일관성 확보.

3. **P1: CAP_FLEX legacy fallback 제거 조건 정의**
   - payroll coverage 임계치/운영 telemetry 기준 충족 시 `cap_flex_enable_legacy_commitment_fallback`를 기본 `False`로 전환.

4. **P2: 버킷 병렬 경로 정리 여부 결정**
   - 이슈 8 최종 목표가 “버킷 완전 제거”인지 “하이브리드 유지”인지 결정 후 불필요 경로 정리.

5. **P2: 레거시 pick expectation 경로 정리 계획 확정**
   - `build_pick_expectations_from_standings()` 삭제 시점(dual-read 종료 시점)과 대체 경로를 문서화.
