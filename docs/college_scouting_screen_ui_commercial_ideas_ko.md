# 대학 리그 스카우팅 화면 상업화 디자인 아이디어 (코드 작업 전 기획안)

## 0) 목표와 제약

### 목표
- 현재의 기능 중심 화면을 **상업 게임 수준의 정보 밀도 + 감성 연출**로 끌어올린다.
- 사용자가 “누구를 어느 스카우터에게 보내야 하는지”를 빠르게 판단하도록 돕는다.
- 화면만 보고도 팀 운영의 전략이 느껴지도록, **결정 중심 UI**로 재구성한다.

### 제약(반드시 준수)
- **새 API 추가 금지.**
- 기존 게임 코드에서 이미 내려주는 조회 데이터만 사용한다.
- 따라서 제안하는 모든 정보 요소는 다음 3가지 중 하나여야 한다.
  1) 기존 원시값 그대로 노출
  2) 기존 원시값의 단순 가공(정렬/그룹/카운트/상태 라벨링)
  3) 기존 원시값 조합으로 만든 파생지표(수학적/규칙 기반, 외부 데이터 불필요)

---

## 1) 현재 화면의 핵심 문제 요약

1. **시각적 우선순위가 약함**
   - 모든 카드가 비슷한 비중으로 보여서 “지금 당장 해야 할 일”이 튀지 않음.
2. **결정 컨텍스트 부족**
   - 스카우터 카드에서 전문성 태그는 보이지만, 배정 후보를 고를 근거(적합도/중복/리스크)가 약함.
3. **상업 게임 톤 부족**
   - 여백/타이포/색 체계가 툴 느낌에 가까워 몰입감이 떨어짐.
4. **작업 흐름 단절**
   - 배정 → 진행 확인 → 리포트 확인의 상태 흐름이 시각적으로 이어지지 않음.

## 2) 화면 구조 재설계(정보 아키텍처)

## A. 상단: “스카우팅 커맨드 바” (한 줄 요약 대시보드)
현재의 단순 텍스트를 다음처럼 확장:
- 활성 배정 수 / 최대 슬롯
- 이번 주 완료 예정 리포트 수
- 미배정 스카우터 수
- 현재 필터 상태(예: 포지션, 학년, 잠재력 구간)

> 전부 기존 데이터 카운트로 계산 가능하며 신규 API 불필요.

### B. 좌측: 스카우터 로스터 패널
- 스카우터 카드를 세로 리스트로 배치(카드 높이 축소, 밀도 증가)
- 각 카드 핵심 정보:
  - 이름 / 전문 분야 / 현재 상태(Idle, Assigned, Report Ready)
  - 현재 배정 대상(있다면 선수명)
  - 전문 태그 2~4개
  - 최근 리포트 신뢰도(기존 점수/등급 데이터가 있다면 노출)

### C. 우측: “후보 선수 워크보드”
- 선택한 스카우터 기준으로 선수 리스트를 보여주고,
- 선수 행마다 “이 스카우터와의 적합 근거”를 배지로 표시
  - 예: `슈팅메커닉`, `수비 스킴`, `모터/컨디션`
- 클릭 시 우측 슬라이드 패널(Quick Preview)
  - 기존 선수 데이터(포지션, 키/몸무게, 핵심 스탯, 잠재력, 리스크 태그)만 조합

### D. 하단 또는 우측 탭: 리포트 인박스
- “새 리포트 N건” 고정 노출
- 스카우터별, 중요도별 정렬
- 클릭 시 리포트 요약 모달(전문 영역 평가 + 총평 + 권장 액션)

## 3) 카드/리스트 컴포넌트 디테일(퀄리티 체감 포인트)

## 3-1. 스카우터 카드(현재 대비 업그레이드)
카드를 3단으로 분할:
1) **헤더**: 이름 / 전문 분야 / 상태 점등(원형 인디케이터)
2) **바디**: 전문 태그 + 현재 배정 대상 미니 프로필
3) **푸터**: 주요 액션 2개
   - 기본: `선수 배정`
   - 보조: `리포트 보기`

#### 상태 컬러 규칙(예시)
- Idle: Slate
- Assigned: Blue
- Report Ready: Emerald
- Risk Alert(의료/부상 위험 높은 대상 배정 시): Amber

> 색은 단순 미감이 아니라 상태 정보 전달 수단으로 사용.

## 3-2. 선수 행(Row) 디자인
- 좌측: 선수명/포지션/학년
- 중앙: 핵심 태그(스카우터 전문성과 맞는 항목 우선)
- 우측: `적합도 점수` + `리스크 뱃지`
- Hover 시 미니 비교 툴팁:
  - “왜 이 선수가 추천되는지”를 2~3줄 자동 문장으로 표시

> 자동 문장도 기존 데이터 기반 규칙 템플릿으로 생성 가능(LLM/API 불필요).

## 3-3. 리포트 카드
- 제목: `[Scout Name] on [Player Name]`
- 핵심 한 줄 요약(긍정/중립/부정 톤)
- 전문 항목별 점수 바(예: Shooting/Defense/IQ)
- CTA: `전체 보기` / `관심 등록` / `드래프트 보드에 반영`

## 4) 게이머 체감을 높이는 “추가 정보” (기존 API만 사용)

아래는 신규 API 없이도 제공 가능한 고체감 요소들:

1. **적합도 점수(0~100)**
   - 스카우터 전문 태그와 선수 태그/스탯을 매칭해 산출
   - 예: Shooting Specialist는 3P, FT, shot_mechanics 가중치 높게

2. **정보 신선도(Freshness)**
   - 마지막 리포트 생성 시점으로 `최신/보통/오래됨` 라벨

3. **중복 스카우팅 경고**
   - 이미 유사 전문 스카우터가 동일 선수 조사 중이면 `중복` 배지

4. **불확실성(Variance) 배지**
   - 보고서 간 평가 편차가 크면 `평가 변동 큼`

5. **리스크 플래그 요약**
   - injury/availability/motor 등 기존 리스크 태그를 상단에 집계 노출

6. **포지션 커버리지 히트맵(간단형)**
   - 최근 N주 동안 포지션별 스카우팅 분포를 막대형으로 표시

> 핵심은 “새로운 데이터”가 아니라 “기존 데이터의 해석 레이어”를 더하는 것.

## 5) 상업용 느낌을 만드는 비주얼 시스템

## 5-1. 톤앤매너
- 다크/라이트 테마 모두 지원하되, 기본은 **다크 스포츠 대시보드** 권장
- 강한 포인트 컬러 1개(팀 컬러) + 상태 컬러 4개로 제한
- 패널 배경은 미세한 노이즈/그라데이션으로 깊이감 부여

## 5-2. 타이포 계층
- H1: 섹션 타이틀(강한 압축 폰트 계열)
- H2: 카드 타이틀
- Body: 숫자/지표 가독성 중심 산세리프
- 숫자 데이터는 tabular-nums 적용(정렬 안정감)

## 5-3. 간격/밀도 규칙
- 8px 그리드 기반
- 카드 내부는 12/16/20 단계만 사용
- 스크롤 영역마다 sticky header를 넣어 정보 맥락 유지

## 5-4. 마이크로 인터랙션
- 배정 성공 시 카드가 살짝 수축→복원 + 상단 토스트
- 리포트 도착 시 해당 스카우터 카드 상단에 맥동 점등 1회
- Hover 강조는 그림자보다 **테두리 발광/색 변화** 위주

---

## 6) “바이브 코딩 티”를 없애는 실전 체크리스트

1. **텍스트 카피 통일**
   - 버튼 문구, 상태 문구, 툴팁 문체를 하나의 스타일 가이드로 통합
2. **정렬 일관성**
   - 버튼 높이/폭, 카드 푸터 정렬, 태그 높이를 전역 토큰으로 고정
3. **의도 있는 비대칭**
   - 모든 카드를 같은 크기로 두지 말고, 중요 영역(워크보드)을 크게
4. **정보 우선순위 확실화**
   - `지금 해야 할 행동`을 항상 1초 안에 찾을 수 있게 강조
5. **로딩/빈 상태 설계**
   - 스켈레톤, 빈 상태 일러스트, 에러 리커버리 CTA 필수

## 7) 즉시 적용 가능한 3단계 개선 로드맵

## Phase 1 (빠른 개선, 1~2일)
- 상단 커맨드 바 추가
- 스카우터 카드 상태 컬러 체계 적용
- 버튼 계층(Primary/Secondary) 정리
- 리포트 인박스 카운트 고정

## Phase 2 (체감 강화, 3~5일)
- 스카우터 선택 기반 후보 워크보드 도입
- 적합도 점수 + 추천 근거 배지 추가
- 중복 스카우팅 경고, 리스크 요약 적용

## Phase 3 (완성도 마감, 1주)
- 모션/전환 디테일
- 다크 테마 고도화
- 타이포/컴포넌트 토큰 정리
- QA: 해상도별 레이아웃 안정화, 키보드 네비게이션

---

## 8) 이 화면에서 반드시 보여줘야 할 핵심 지표(최소 세트)

- 활성 배정 수 / 최대 배정 수
- 미배정 스카우터 수
- 완료 대기 리포트 수
- 스카우터별 현재 타겟 선수
- 타겟 선수의 적합도(스카우터 기준)
- 리스크 플래그 보유 여부

이 6가지만 명확히 보여도 의사결정 속도와 만족도가 크게 오른다.

---

## 9) 최종 디자인 컨셉 한 줄 제안

**“Front Office War Room”**: 
감성은 차갑고 프로페셔널하게, 의사결정은 빠르고 공격적으로.
사용자는 스카우터를 ‘카드 클릭’이 아니라 ‘전략 자원 배치’로 체감해야 한다.

      ↓
