# 프리셋 공격 스킴 설계안 v2 (유저 선택 분리)

이 문서는 `프리셋_스킴_구현_사전작업_유저선택_분리.md`의 원칙(고정 틀 + 유저 오버레이)을 따르면서,
공격 프리셋(`Preset_Offense`)의 유저 선택 항목을 구체화한 설계안이다.

---

## 1) 설계 목표

- 유저가 **큰 전술 방향**을 먼저 정하고, 그 안에서 **세부 빈도/성향**을 조정할 수 있게 한다.
- 엔진 관점에서는 모든 입력을 런타임 overlay로 병합하고 clamp/normalize하여 안정성을 보장한다.
- 액션/아웃컴 매핑은 반드시 `action_outcome_priors.py`의 실제 키 범위 안에서만 허용한다.

---

## 2) Base Action 빈도 UI/데이터 구조

### 2-1. 상위 카테고리(대분류)

유저는 먼저 아래 7개 상위 카테고리의 볼륨을 조정한다.

1. `픽앤롤`
2. `드라이브`
3. `속공`
4. `아이솔`
5. `패스 빈도`
6. `오프볼 액션`
7. `포스트업`

### 2-2. 상위 카테고리 → 엔진 액션 매핑

- `픽앤롤` → `PnR`, `PnP` (상위 볼륨 내에서 하위 비중 세부 조정 가능)
- `드라이브` → `Drive` (1:1)
- `속공` → `TransitionEarly` (1:1)
- `아이솔` → `ISO` (1:1)
- `패스 빈도` → `Kickout`, `ExtraPass` (동일 방향 동시 증감, 개별 편집 불가)
- `오프볼 액션` → `Cut`, `SpotUp`, `DHO` (상위 볼륨 내에서 하위 비중 세부 조정 가능)
- `포스트업` → `PostUp` (1:1)

### 2-3. 정규화 규칙

- 상위 카테고리 값 변경 후 전체 base action은 합계 100(또는 내부 기준합)으로 정규화.
- 하위 슬라이더가 있는 카테고리(`픽앤롤`, `오프볼 액션`)는:
  - 상위 볼륨 = 하위 항목 분배 총량
  - 하위 비중도 카테고리 내 합 100으로 정규화
- `패스 빈도`는 `Kickout`과 `ExtraPass`가 같은 방향으로 동일률 증감.

## 3) Action별 Outcome 성향 (수정 반영본)

아래는 유저가 조정 가능한 액션별 outcome 그룹이다.

> 원칙: 존재하지 않는 outcome 키는 UI에서 노출하지 않는다.

### 3-1. PnR

- `SHOT_RIM_*`(통합 그룹)
- `SHOT_3_OD`
- `PASS_SHORTROLL`
- `SHOT_TOUCH_FLOATER`
- `SHOT_MID_PU`

### 3-2. PnP

- `SHOT_RIM_*`(통합 그룹)
- `SHOT_3_OD`
- `SHOT_3_CS`
- `SHOT_TOUCH_FLOATER`
- `SHOT_MID_PU`

### 3-3. TransitionEarly

- `SHOT_RIM_*`(통합 그룹)
- `SHOT_3_OD`
- `SHOT_3_CS`
- `SHOT_TOUCH_FLOATER`

> v2 변경: `SHOT_MID_PU` 제거.

### 3-4. Drive

- `SHOT_RIM_*`(통합 그룹)
- `PASS_KICKOUT`
- `SHOT_MID_PU`

> v2 변경: `SHOT_3_CS` 제거.

### 3-5. ISO

- `SHOT_RIM_LAYUP + SHOT_RIM_DUNK`(통합 그룹)
- `SHOT_RIM_CONTACT`
- `SHOT_TOUCH_FLOATER`
- `SHOT_MID_PU`
- `PASS_KICKOUT`
- `SHOT_3_OD`

### 3-6. Cut

- `SHOT_RIM_*`(통합 그룹)
- `PASS_EXTRA + PASS_KICKOUT`(통합 그룹)

### 3-7. PostUp

- `SHOT_POST`
- `SHOT_MID_PU`
- `PASS_*`(통합 그룹: `PASS_EXTRA`, `PASS_KICKOUT`, `PASS_SKIP`)

## 4) 신규 전역 성향(스킴 전체 적용)

아래 2개는 특정 action이 아니라 **공격 스킴 전체**에 적용되는 전역 슬라이더다.

## 4-1. 파울 유도 성향 (`foul_draw_bias`)

- 슬라이더 ↑:
  - 전체 `FOUL_DRAW_*` outcome 총량 증가
  - 전체 `SHOT_*` outcome 총량 감소
- 슬라이더 ↓:
  - 위의 반대 동작

### 적용 규칙

- 대상 집합
  - 증가군: `FOUL_DRAW_*`
  - 감소군: `SHOT_*`
- 총량 보존 원칙
  - 증가군 총증가량 = 감소군 총감소량
- 균등 증감 원칙(집합 내부)
  - 같은 집합 내부에서는 각 outcome이 동일한 절대량(또는 동일한 비율)로 조정
- 개수 보정
  - `FOUL_DRAW_*` 개수 < `SHOT_*` 개수이므로,
  - **개별 FOUL 항목의 단위 증감폭은 개별 SHOT 항목보다 크게** 설정

## 4-2. 위험 감수 성향 (`risk_taking_bias`)

- 슬라이더 ↑:
  - 전체 `TO_*` outcome 총량 증가
  - 전체 `RESET_*` outcome 총량 감소
- 슬라이더 ↓:
  - 위의 반대 동작

### 적용 규칙

- 대상 집합
  - 증가군: `TO_*`
  - 감소군: `RESET_*`
- 총량 보존 원칙
  - 증가군 총증가량 = 감소군 총감소량
- 균등 증감 원칙(집합 내부)
  - 같은 집합 내부 outcome은 동일 룰로 증감
- 개수 보정
  - `TO_*`와 `RESET_*` 개수가 다를 수 있으므로,
  - **개별 outcome 단위 증감폭은 집합 크기 역비례 방식으로 자동 보정**

## 5) 권장 계산 순서(런타임)

1. `Preset_Offense` 기본 테이블 로드(안전 중립값)
2. base action 상위/하위 슬라이더 반영
3. action별 outcome 성향 반영
4. 전역 성향 반영
   - `foul_draw_bias`: `FOUL_DRAW_* ↔ SHOT_*`
   - `risk_taking_bias`: `TO_* ↔ RESET_*`
5. clamp
   - 음수 방지
   - 허용 상한 제한
6. normalize
   - action 분포 합 정규화
   - action 내부 outcome 분포 합 정규화

## 7) 구현 시 주의사항

- 전역 성향은 action별 성향 적용 이후에 적용해야 의도한 전체 이동량이 명확하다.
- 전역 성향 적용 후 음수/과도값이 발생하면 clamp 후 재정규화한다.
- 집합 크기가 다를 때는 “총량 동일 이동”을 만족하도록 항목별 단위 이동량을 자동 계산한다.
- 존재하지 않는 액션/아웃컴 키는 저장 시 무시하고, 디버그 로그에 남긴다.

## 8) v2 변경 요약

- `Drive` outcome에서 `SHOT_3_CS` 제거.
- `TransitionEarly` outcome에서 `SHOT_MID_PU` 제거.
- 스킴 전역 슬라이더 2종 추가:
  - `파울 유도 성향` (`FOUL_DRAW_* ↔ SHOT_*`)
  - `위험 감수 성향` (`TO_* ↔ RESET_*`)
- 두 전역 슬라이더 모두 “총량 동일 이동 + 집합 크기 차이 보정” 규칙을 명시.
