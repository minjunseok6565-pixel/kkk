# 공격/수비 스킴 신규 추가 체크리스트

목표: 신규 스킴이 기존 스킴들과 **동일한 입력 경로(UI/API)**, **동일한 엔진 반영 지점(행동 선택/결과 확률/품질 보정)**, **동일한 학습/표시 경로**에 영향을 주도록 한다.

---

## 1) 공격 스킴 추가 시 필수 반영 항목

1. **프론트 스킴 목록 노출**
   - 파일: `static/js/core/constants/tactics.js`
   - `TACTICS_OFFENSE_SCHEMES`에 `{ key, label }` 추가

2. **행동 선택 분포(베이스 가중치)**
   - 파일: `matchengine_v3/profiles_data/off_scheme_action_weights.py`
   - 추가 위치: `OFF_SCHEME_ACTION_WEIGHTS[<신규공격스킴>] = {...}`
   - 정확한 데이터 형태:
     - key: `base_action` 문자열
     - value: 가중치 숫자(int/float)
   - 현재 엔진에서 실제 사용하는 `base_action` 키 전체(누락 없이 맞추는 것을 권장):
     - `PnR`, `PnP`, `TransitionEarly`, `Drive`, `ISO`, `Kickout`, `ExtraPass`, `SpotUp`, `Cut`, `PostUp`, `DHO`, `HornsSet`
   - 참고: 미추가 시 엔진에서 fallback 스킴(`Spread_HeavyPnR`)로 동작함

3. **결과 확률 보정(공격 스킴 multiplier)**
   - 파일: `matchengine_v3/profiles_data/offense_scheme_mult.py`
   - 추가 위치: `OFFENSE_SCHEME_MULT[<신규공격스킴>] = {...}`
   - 정확한 데이터 형태:
     - 1단: `base_action` 키
     - 2단: `outcome` 키
     - 3단: multiplier 숫자
     - 예시 형태:
       ```python
       OFFENSE_SCHEME_MULT["My_New_Off_Scheme"] = {
         "PnR": {
           "PASS_KICKOUT": 1.05,
           "SHOT_3_OD": 1.08,
         },
         "Drive": {
           "SHOT_RIM_LAYUP": 1.06,
         },
       }
       ```
   - 현재 테이블에서 사용 중인 `base_action` 키:
     - `PnR`, `PnP`, `TransitionEarly`, `Drive`, `ISO`, `Kickout`, `ExtraPass`, `Cut`, `PostUp`, `DHO`, `HornsSet`
   - 현재 테이블에서 사용 중인 `outcome` 키 전체:
     - `PASS_KICKOUT`, `PASS_SHORTROLL`, `PASS_EXTRA`, `PASS_SKIP`
     - `SHOT_RIM_LAYUP`, `SHOT_RIM_DUNK`, `SHOT_RIM_CONTACT`
     - `SHOT_TOUCH_FLOATER`, `SHOT_MID_PU`, `SHOT_MID_CS`
     - `SHOT_3_OD`, `SHOT_3_CS`, `SHOT_POST`
     - `FOUL_DRAW_RIM`, `FOUL_DRAW_POST`
     - `TO_HANDLE_LOSS`, `TO_CHARGE`
     - `RESET_HUB`, `RESET_RESREEN`, `RESET_REDO_DHO`, `RESET_POST_OUT`

4. **샷다이어트(Shot Diet) 지원 스킴 등록**
   - 파일: `matchengine_v3/validation.py`
   - `SHOT_DIET_SUPPORTED_OFFENSE_SCHEMES`에 신규 키 추가
   - 없으면 validation 단계에서 `Spread_HeavyPnR` fallback/에러 발생 가능

5. **샷다이어트 세부 테이블 동기화**
   - 파일: `matchengine_v3/shot_diet_data.py`
   - 아래 4개에 동일한 신규 스킴 키 추가
     - `WEIGHTS_TACTIC_ACTION`
     - `WEIGHTS_TACTIC_OUTCOME_DELTA`
     - `TACTIC_ALPHA`
     - `SCREENER_ROLE_PRIORITY`
   - 5-1) `WEIGHTS_TACTIC_ACTION[스킴][base_action][feature] = weight`
     - `base_action` 사용 키: `PnR`, `PnP`, `TransitionEarly`, `Drive`, `ISO`, `Kickout`, `ExtraPass`, `SpotUp`, `Cut`, `PostUp`, `DHO`, `HornsSet`
     - `feature` 사용 키: `BH_BALL_SECURITY`, `BH_DRIVE_PRESSURE`, `BH_PASS_CREATION`, `BH_PNR`, `BH_PULLUP_THREAT`, `SC_POP_THREAT`, `SC_ROLL_FINISH`, `SC_SCREEN_QUALITY`, `SC_SHORTROLL_PLAY`, `TEAM_CATCH3_QUALITY`, `TEAM_CUTTING`, `TEAM_EXTRA_PASS`, `TEAM_PACE`, `TEAM_POST_GRAVITY`, `TEAM_SPACING`, `D_POA`, `D_HELP_CLOSEOUT`, `D_RIM_PROTECT`, `D_STEAL_PRESS`, `D_POST`, `D_DREB`
   - 5-2) `WEIGHTS_TACTIC_OUTCOME_DELTA[스킴][base_action][outcome][feature] = delta_weight`
     - `base_action` 사용 키: `PnR`, `TransitionEarly`, `Drive`, `Kickout`, `ExtraPass`, `SpotUp`, `Cut`, `PostUp`, `DHO`, `HornsSet`
     - `outcome` 사용 키: `PASS_KICKOUT`, `PASS_SHORTROLL`, `PASS_EXTRA`, `SHOT_RIM_LAYUP`, `SHOT_3_CS`, `SHOT_3_OD`, `SHOT_MID_PU`, `SHOT_POST`, `FOUL_DRAW_RIM`, `TO_HANDLE_LOSS`, `TO_CHARGE`, `RESET_REDO_DHO`, `RESET_POST_OUT`
     - `feature` 사용 키: `BH_BALL_SECURITY`, `BH_DRIVE_PRESSURE`, `BH_FOUL_PRESSURE`, `BH_PULLUP_THREAT`, `SC_SCREEN_QUALITY`, `SC_SHORTROLL_PLAY`, `TEAM_CATCH3_QUALITY`, `TEAM_CUTTING`, `TEAM_EXTRA_PASS`, `TEAM_PACE`, `TEAM_POST_GRAVITY`, `TEAM_SPACING`, `D_RIM_PROTECT`
   - 5-3) `TACTIC_ALPHA[스킴] = (alpha_action, alpha_outcome)`
   - 5-4) `SCREENER_ROLE_PRIORITY[스킴] = [role1, role2, ...]`
   - 별칭 필요 시 `SCHEME_ALIASES`에도 추가

6. **팀 스타일 보정값**
   - 파일: `matchengine_v3/possession/team_style.py`
   - 추가 위치: `scheme_mods[<신규공격스킴>] = {...}`
   - 정확한 데이터 형태:
     - key: `three_bias`, `rim_bias`, `tempo_mult` 중 필요한 항목
     - value: multiplier 숫자
   - 예시:
     ```python
     "My_New_Off_Scheme": {
       "three_bias": 1.02,
       "tempo_mult": 1.01,
     }
     ```
   - 필수는 아니지만, 기존 일부 스킴은 여기서 추가 성향을 받음

## 2) 수비 스킴 추가 시 필수 반영 항목

1. **프론트 스킴 목록 + 포지션별 수비 롤 매핑**
   - 파일: `static/js/core/constants/tactics.js`
   - `TACTICS_DEFENSE_SCHEMES`에 `{ key, label }` 추가
   - `TACTICS_DEFENSE_ROLE_BY_SCHEME[<신규수비스킴>]`에 5개 롤 시퀀스 추가

2. **수비 스킴 canonical/alias 등록**
   - 파일: `matchengine_v3/tactics.py`
   - `_CANON_DEFENSE_SCHEMES`에 canonical 키 추가
   - `_DEFENSE_SCHEME_ALIAS_NORM`에 alias(영문/한글/UI 표기) 추가

3. **결과 확률 보정(수비 스킴 multiplier)**
   - 파일: `matchengine_v3/profiles_data/defense_scheme_mult.py`
   - 추가 위치: `DEFENSE_SCHEME_MULT[<신규수비스킴>] = {...}`
   - 정확한 데이터 형태:
     - 1단: `base_action`
     - 2단: `outcome`
     - 3단: multiplier 숫자
   - 현재 테이블에서 사용 중인 `base_action` 키:
     - `PnR`, `PnP`, `Drive`, `ISO`, `Kickout`, `SpotUp`, `PostUp`, `DHO`, `HornsSet`
   - 현재 테이블에서 사용 중인 `outcome` 키 전체:
     - `PASS_KICKOUT`, `PASS_SHORTROLL`, `PASS_EXTRA`, `PASS_SKIP`
     - `SHOT_RIM_LAYUP`, `SHOT_RIM_DUNK`, `SHOT_RIM_CONTACT`
     - `SHOT_TOUCH_FLOATER`, `SHOT_MID_PU`, `SHOT_MID_CS`
     - `SHOT_3_OD`, `SHOT_3_CS`, `SHOT_POST`
     - `FOUL_DRAW_POST`, `FOUL_REACH_TRAP`
     - `TO_HANDLE_LOSS`, `TO_CHARGE`, `TO_BAD_PASS`, `TO_SHOT_CLOCK`
     - `RESET_HUB`, `RESET_RESREEN`, `RESET_REDO_DHO`

4. **포제션 문맥 기반 도움수비 baseline 반영**
   - 파일: `matchengine_v3/possession/tactics_ctx.py`
   - `_SCHEME_HELP_BASELINE[<신규수비스킴>]` 추가

5. **수비 메타 보정 규칙(행동/결과 prior) 추가**
   - 파일: `matchengine_v3/era.py`
   - 5-1) `DEFENSE_META_PARAMS["defense_meta_action_mult_tables"][<신규수비스킴>]`
     - 정확한 형태: `{base_action: multiplier}`
     - 현재 테이블에서 사용 중인 `base_action` 키:
       - `PnR`, `Drive`, `Kickout`, `SpotUp`, `PostUp`, `DHO`, `ExtraPass`, `HornsSet`, `Cut`
   - 5-2) `DEFENSE_META_PARAMS["defense_meta_priors_rules"][<신규수비스킴>]`
     - 정확한 형태: `[{...rule...}, {...rule...}]`
     - rule에서 현재 사용되는 필드:
       - `key` (대상 outcome)
       - `mult` (곱 보정)
       - `min` (최소치 보정)
       - `require_base_action` (특정 base_action일 때만 적용)
     - `require_base_action` 현재 사용값: `PnR`
     - `key`(대상 outcome) 현재 사용값:
       - `PASS_KICKOUT`, `PASS_SHORTROLL`, `PASS_EXTRA`
       - `SHOT_RIM_LAYUP`, `SHOT_RIM_DUNK`, `SHOT_RIM_CONTACT`
       - `SHOT_MID_PU`, `SHOT_3_OD`, `SHOT_3_CS`, `SHOT_POST`
       - `TO_HANDLE_LOSS`

6. **품질(QUALITY) 시스템 테이블 동기화**
   - 파일: `matchengine_v3/quality_data/*`
   - 아래 4개 모두에 신규 스킴 키/별칭 추가
     - `scheme_base_outcome_labels.py` (`SCHEME_BASE_OUTCOME_LABELS`)
     - `role_stat_profiles.py` (`ROLE_STAT_PROFILES`)
     - `group_scheme_role_weights.py` (`GROUP_SCHEME_ROLE_WEIGHTS`)
     - `scheme_aliases.py` (`SCHEME_ALIASES`)
   - 6-1) `SCHEME_BASE_OUTCOME_LABELS[스킴][base_action][outcome] = quality_label`
     - `base_action` 사용 키: `PnR`, `PnP`, `TransitionEarly`, `Drive`, `ISO`, `Kickout`, `ExtraPass`, `SpotUp`, `Cut`, `PostUp`, `DHO`, `HornsSet`
     - `outcome` 사용 키: `PASS_KICKOUT`, `PASS_SHORTROLL`, `PASS_EXTRA`, `PASS_SKIP`, `SHOT_RIM_LAYUP`, `SHOT_RIM_DUNK`, `SHOT_RIM_CONTACT`, `SHOT_TOUCH_FLOATER`, `SHOT_MID_PU`, `SHOT_MID_CS`, `SHOT_3_OD`, `SHOT_3_CS`, `SHOT_POST`, `FOUL_DRAW_RIM`, `FOUL_DRAW_POST`, `FOUL_DRAW_JUMPER`, `TO_HANDLE_LOSS`, `TO_CHARGE`, `RESET_HUB`, `RESET_RESREEN`, `RESET_REDO_DHO`
     - `quality_label` 사용값: `weak`, `neutral`, `Neutral`, `tight`, `tough`
       - 주의: 현재 데이터에 `neutral`/`Neutral` 혼재
   - 6-2) `ROLE_STAT_PROFILES[스킴][def_role][stat_key] = weight`
     - `def_role` 사용 키 전체:
       - `PnR_POA_Defender`, `PnR_Cover_Big_Drop`, `Lowman_Helper`, `Nail_Helper`, `Weakside_Rotator`
       - `PnR_POA_Switch`, `PnR_Cover_Big_Switch`, `Switch_Wing_Strong`, `Switch_Wing_Weak`, `Backline_Anchor`
       - `PnR_POA_Switch_1_4`, `PnR_Cover_Big_Switch_1_4`, `Switch_Wing_Strong_1_4`, `Switch_Wing_Weak_1_4`
       - `PnR_Cover_Big_HedgeRecover`, `PnR_POA_Blitz`, `PnR_Cover_Big_Blitz`
       - `PnR_POA_AtTheLevel`, `PnR_Cover_Big_AtTheLevel`
       - `Zone_Top_Left`, `Zone_Top_Right`, `Zone_Bottom_Left`, `Zone_Bottom_Right`, `Zone_Bottom_Center`
     - `stat_key` 사용 키: `DEF_POA`, `DEF_HELP`, `DEF_RIM`, `DEF_POST`, `DEF_STEAL`, `REB_DR`, `PHYSICAL`, `ENDURANCE`
   - 6-3) `GROUP_SCHEME_ROLE_WEIGHTS[group_id][스킴][def_role] = weight`
     - `group_id` 사용 키: `rim_finish`, `pullup`, `catch_shoot`, `paint_non_rim`, `pass_kickout`, `pass_shortroll`, `pass_skip`, `pass_extra`, `to_handle_loss`, `to_charge`
     - `def_role`은 `ROLE_STAT_PROFILES`와 동일 role set을 맞출 것
   - 6-4) `SCHEME_ALIASES[alias] = canonical_scheme`
   - 이 블록을 빼먹으면 resolve 단계의 quality delta 반영 강도가 기존 스킴 대비 불완전해질 수 있음

## 3) 공통 체크(공격/수비 모두)

1. **UI/서버 payload 키 일치 확인**
   - 파일: `app/api/routes/tactics.py`
   - `offenseScheme/defenseScheme` ↔ `offense_scheme/defense_scheme` 직렬화 구간 확인

2. **테스트 케이스 보강**
   - 최소 권장
     - `tests/test_tactics_api.py`: 신규 키 round-trip
     - (가능 시) 시뮬레이션 smoke 테스트 1개: 신규 공격/수비 스킴으로 possession 진행

3. **기존 fallback 의존 여부 점검**
   - fallback으로 "돌아가긴 하는" 상태를 목표로 하지 말고,
   - 위 체크리스트 모든 항목에 신규 스킴 키가 존재하도록 맞추는 것을 완료 기준으로 삼는다.

## 4) 빠른 점검용 grep

```bash
rg -n "<신규공격스킴키>|<신규수비스킴키>" \
  static/js/core/constants/tactics.js \
  matchengine_v3/profiles_data/off_scheme_action_weights.py \
  matchengine_v3/profiles_data/offense_scheme_mult.py \
  matchengine_v3/profiles_data/defense_scheme_mult.py \
  matchengine_v3/shot_diet_data.py \
  matchengine_v3/validation.py \
  matchengine_v3/tactics.py \
  matchengine_v3/possession/team_style.py \
  matchengine_v3/possession/tactics_ctx.py \
  matchengine_v3/era.py \
  matchengine_v3/quality_data
```

위 결과에서 신규 키가 한 군데라도 비면, 기존 스킴과 "같은 영향 범위"를 갖지 못할 가능성이 높다.
