## 1-1. `matchengine_v3/player_behavior.py` (신규)

### 목적
- C13 세부행동 관련 SSOT 모듈
- 입력 검증 + 기본값 + 역할 잠금 강제 + 확률 보정 유틸 제공

### 상수/타입

```python
# behavior keys
BEHAVIOR_KEYS_ONBALL = (
    "ON_PULLUP", "ON_RIM_ATTACK", "ON_PNR_BH", "ON_POST_TOUCH", "ON_PASS_FIRST", "onball_isolation",
)
BEHAVIOR_KEYS_OFFBALL = (
    "OFF_SPOTUP_SPACE", "OFF_CUT", "OFF_OFFSCREEN", "OFF_SCREEN_ROLE", "offball_crash_off_reb",
)
BEHAVIOR_KEYS_ALL = BEHAVIOR_KEYS_ONBALL + BEHAVIOR_KEYS_OFFBALL

LEVEL_KEYS = {"LOW", "MID", "HIGH"}
SCREEN_ROLE_KEYS = {"ROLL", "POP", "AUTO"}

LEVEL_MULT = {"LOW": 0.92, "MID": 1.00, "HIGH": 1.08}
ORB_LEVEL_MULT = {"LOW": 0.90, "MID": 1.00, "HIGH": 1.10}  # ORB만 별도
```

### 역할 잠금 테이블
- 문서(`C13_역할별_세부행동_선택규칙.md`)를 코드화해 `LOCK_RULES_BY_ROLE` 선언
- 형태:

```python
LOCK_RULES_BY_ROLE = {
    "Shot_Creator": {"ON_PULLUP": "HIGH", "onball_isolation": "HIGH"},
    "Roll_Man": {"OFF_SCREEN_ROLE": "ROLL", "offball_crash_off_reb": "HIGH"},
    ...
}
```

### 핵심 함수 시그니처

```python

    """행동 설정을 action 확률에 반영 (정규화 포함)."""

    pri: Dict[str, float],
    behavior: Mapping[str, str],
) -> Dict[str, float]:
    """행동 설정을 outcome priors에 반영 (정규화 포함)."""


def orb_behavior_multiplier(behavior: Mapping[str, str]) -> float:
    """offball_crash_off_reb 기반 ORB 개인 배율 반환."""
```

### sanitize 규칙(고정)
1. `raw_map`이 dict가 아니면 `{}` 반환
2. pid key/string 정리 (`str(pid).strip()`)
3. behavior key whitelist만 유지
4. value는 key별 허용값으로 정규화
   - 강도형 키: 미허용 시 `MID`
   - `OFF_SCREEN_ROLE`: 미허용 시 `AUTO`
5. `role_by_pid`에서 역할 확인 후 잠금값 강제 적용
6. 결과는 `Dict[pid, Dict[key, val]]`

### action 보정 규칙(1차안)
- `onball_isolation`: `ISO +`, `ExtraPass -`
- `ON_PNR_BH`: `PnR/PnP +`
- `ON_RIM_ATTACK`: `Drive +`, `SpotUp -`
- `ON_POST_TOUCH`: `PostUp +`
- `ON_PASS_FIRST`: `ExtraPass/Kickout +`, `ISO -`
- 곱연산 후 clamp `[0.82, 1.22]`, 마지막 normalize

### outcome 보정 규칙(1차안)
- `ON_PULLUP HIGH`: `SHOT_3_OD/SHOT_MID_PU +`, `PASS_KICKOUT -`
- `ON_RIM_ATTACK HIGH`: `SHOT_RIM_* / FOUL_DRAW_RIM +`, `SHOT_MID_PU -`
- `ON_PASS_FIRST HIGH`: `PASS_* / RESET_HUB +`, `SHOT_* 일부 -`
- `OFF_SCREEN_ROLE=ROLL`: `PASS_SHORTROLL/SHOT_RIM_* +`
- `OFF_SCREEN_ROLE=POP`: `SHOT_3_CS/SHOT_MID_CS +`
- 곱연산 후 clamp `[0.80, 1.25]`, 마지막 normalize

## 2-1. `app/api/routes/tactics.py`

### 수정 목적
- tactics PUT 시 `PLAYER_BEHAVIOR_BY_PID`를 엔진 저장 전에 sanitize
- tactics GET 시도 UI가 그대로 읽을 수 있게 context 포함 유지

### 구체 수정
1. import 추가
```python
from matchengine_v3.player_behavior import sanitize_player_behavior_by_pid
```

2. `_to_engine_tactics(payload)` 내부
- `context = dict(raw.get("context") or {})`
- `off_by_pid` 생성 후:
```python
ctx_beh = context.get("PLAYER_BEHAVIOR_BY_PID")
context["PLAYER_BEHAVIOR_BY_PID"] = sanitize_player_behavior_by_pid(ctx_beh, off_by_pid)
```
- return payload의 `context`는 sanitize된 context 사용

3. 이미 엔진형 payload인 경우(`if "offense_scheme" in raw ...`)
- `rotation_offense_role_by_pid`를 읽어 같은 sanitize 적용

### 주의
- API 레이어에서는 예외를 던지지 않고 가능한 한 normalize + fallback
- strict validation은 `matchengine_v3/validation.py`에서 최종 보강

### 수정 목적
- possession 실행 시 실제 action/outcome에 behavior 반영

2. action 확률 계산부 직후



## 6) 구현 시 주의사항

- 기존 전술 multipliers와 behavior multipliers는 중복 과증폭을 피하기 위해 서로 clamp를 분리한다.
- normalize 이전에 0 또는 음수가 되지 않게 최소값 `1e-6`을 유지한다.
- behavior 미설정 pid는 항상 neutral(`MID`/`AUTO`)로 취급한다.
- 리그 저장데이터 하위호환을 위해 behavior 키가 없어도 정상 동작해야 한다.
