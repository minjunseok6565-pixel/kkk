# Preset Defense 상세 설정 UI 리팩터링 계획 (오버레이 모달 + 액션별 세부 확장)

## 1) 요청 계획 검토 결과

요청하신 방향은 현재 코드 구조와 잘 맞고, UX 관점에서도 타당합니다.

- **방향성 타당**: `프리셋 수비 상세 설정` 버튼 클릭 시 독립 오버레이 모달에서만 편집하도록 고정하면, 전술 탭 본문 흐름(로스터/라인업)과 상세 편집 맥락이 분리되어 사용성이 좋아집니다.
- **정보 구조 개선 타당**: 현재처럼 `액션 버짓`과 `액션별 묶음 선택`을 분리한 2개 섹션보다, “액션 버짓 항목 클릭 → 해당 액션 세부 묶음 선택 확장” 구조가 인지 부하를 줄입니다.
- **확장/접기 기본 정책 타당**: 모든 항목 기본 닫힘 + 클릭 시 개별 확장 + 다중 동시 확장 허용은 비교 편집 시나리오(2~3개 액션을 동시에 맞추는 작업)에 유리합니다.
- **전역 압박 배치 타당**: 모달 하단 고정 영역(또는 마지막 섹션)에 전역 압박 슬라이더를 두면 “액션별 미세 조정 후 전체 톤 조정” 흐름이 자연스럽습니다.

---

## 2) 현재 구조 기준 핵심 이슈(원인 진단)

### A. 모달이 오버레이처럼 보이지 않는 원인 후보
- 전술 화면 모달의 기반 스타일은 `#tactics-screen .modal` 스코프에 정의되어 있습니다.
- 그런데 `preset-offense-modal`, `preset-defense-modal` DOM이 `#tactics-screen` 블록 바깥에 배치되어 있으면, 해당 스코프 스타일이 적용되지 않아 일반 블록처럼 흐름에 끼어들 수 있습니다.
- 지금 보고 계신 현상(로스터 목록 아래로 열림)은 이 스코프 불일치와 정확히 일치하는 증상입니다.

### B. 수비 프리셋 렌더링 구조의 한계
- 현재 `presetDefenseModal.js`는 렌더 단위를
  1) 액션 버짓 전체 그리드
  2) 액션별 묶음 선택 전체 그리드
  3) 전역 압박
  으로 분리해 그리고 있습니다.
- 따라서 요청하신 “버짓 항목을 클릭하면 해당 항목의 세부 묶음 설정이 열리는 구조”로 바꾸려면 **DOM 구조/상태/이벤트를 함께 재설계**해야 합니다.

## 3) 목표 UI 구조(합의안)

## 3-1. 모달 레이아웃
1. 헤더: 제목 + 닫기
2. 본문:
   - 액션 리스트(9개): 각 행에 `액션명`, `버짓 슬라이더`, `값`, `세부 설정 열기/닫기`
   - 각 액션 행 아래 확장 영역(기본 닫힘):
     - 묶음 선택 A/중립/B
     - 필요 시 억제 강도(normal/strong)
3. 하단:
   - 전역 압박 슬라이더
   - 초기화/적용 버튼

## 3-2. 인터랙션 규칙
- 최초 진입 시 9개 액션의 세부 영역은 모두 닫힘
- 액션 행 클릭(또는 토글 버튼)으로 열림/닫힘
- 여러 액션을 동시에 열 수 있음(아코디언 단일 오픈 아님)
- 버짓 변경/묶음 변경/압박 변경 시 즉시 draft 반영 + validation

## 4) 파일별 작업 계획

- `preset-defense-modal` DOM이 `#tactics-screen` 내부 스코프 스타일을 확실히 받도록 위치/중첩 재점검
- 접근성 속성 유지(`role="dialog"`, `aria-modal`, `aria-labelledby`, `aria-describedby`)

- 모달이 화면 중앙 오버레이로 뜨는지
- 배경 스크롤/클릭 차단(backdrop) 정상 동작

## 4-2. `static/css/screens/tactics.css`
### 작업
- 수비 프리셋 섹션 스타일을 “분리형 2섹션(버짓/정책)” 기준에서 “액션별 확장형 카드/행” 기준으로 재작성
- 신규 권장 클래스 예시
  - `.preset-defense-action-list`
  - `.preset-defense-action-item`
  - `.preset-defense-action-head`
  - `.preset-defense-action-detail`
  - `.is-expanded`(열림 상태)
- 전역 압박 슬라이더 영역을 모달 하단에서 시각적으로 구분

### 체크 포인트

- 모바일(840px 이하)에서 슬라이더/버튼 줄바꿈이 깨지지 않는지

## 4-3. `static/js/features/tactics/presetDefenseModal.js`
### 작업
1. **상태 추가**
   - 예: `expandedActions = new Set()`
   - 기본값은 빈 Set(전부 닫힘)
2. **렌더링 구조 변경**
   - `_renderBudgetRow`, `_renderPolicyRow` 분리 렌더를 통합해
   - `_renderActionItem(action, draft, isExpanded)` 형태로 전환
3. **이벤트 바인딩 변경**
   - `data-defense-toggle` 클릭으로 액션별 확장/접기
   - 기존 `data-defense-side`, `data-defense-level`, `data-defense-budget`는 액션 카드 내부에서 동일 처리
4. **재렌더 안정화**
   - 슬라이더 입력 시 전체 `innerHTML` 재생성 구조 유지 시 포커스 점프/스크롤 점프 최소화 보완
   - 가능하면 이벤트 위임 패턴으로 리스너 재부착 비용 감소
5. **검증/오류 메시지 정책 유지**
   - `_validateDraft` 결과를 기존처럼 적용하되, 경고 문구를 새 UI 용어에 맞게 조정

### 체크 포인트
- 다중 확장 동작 보장
- reset/apply 후에도 의도한 확장 상태 정책(초기화 시 전부 닫힘 권장) 유지
- ESC/Tab 포커스 트랩 계속 정상 동작

## 5) 구현 순서 제안 (작업 단위)

1. **오버레이 보장 먼저 해결**
   - HTML 구조/스코프 문제 해결 → 모달 위치 이슈 재현 종료 확인
2. **수비 모달 마크업/스타일 리팩터링**
   - 액션별 확장형 UI 뼈대 + 하단 전역 압박 배치
3. **JS 상태/이벤트 리팩터링**
   - 확장 상태 Set + 토글 이벤트 + 기존 입력 이벤트 연결
4. **회귀 점검**
   - 열기/닫기, 초기화/적용, 전술 저장, 재진입 복원, 모바일 뷰

---

## 6) 수용 기준(Definition of Done)

- `프리셋 수비 상세 설정` 버튼 클릭 시, 전술 탭 본문이 아닌 **화면 오버레이 모달**로 노출
- 모달 기본 진입 시 모든 액션 세부 설정 닫힘
- 임의의 여러 액션 세부 설정을 동시에 열 수 있음
- 각 액션에서 버짓 + 묶음 선택(및 강도)이 한 흐름에서 편집 가능
- 모달 하단에 전역 압박 슬라이더 존재
- 적용 시 기존 저장 파이프라인(compile/merge/serde)과 충돌 없음

## 8) 추가 권장(선택)

- 액션 항목 우측에 “현재 선택 요약 배지”(예: `A 강억제`, `중립`)를 두면 닫힌 상태에서도 빠르게 전체 상태 파악 가능
- 모달 재오픈 시 확장 상태는 초기화(전부 닫힘)하는 정책이 요구사항과 일치하며, 예측 가능성이 높음
