## 2) 공통 세부 설정 항목(역할군별)

### A. `POA_DEFENDER`
1. 스크린 대처 방식: `파이트 스루 / 언더 / 스위치 / ICE`
2. 핸들러 압박 강도: `강하게 / 보통 / 약하게`
3. 스틸 시도 성향: `적극적 / 균형 / 적게 시도`
4. 압박 지점: `풀코트 / 하프라인 / 세트된 후`

### B. `WING_DEFENDER`
1. 헬프 강도: `강하게 / 보통 / 약하게`
2. 리바운드 참여도: `적극 참여 / 보통 / 백코트 우선`
3. 스위치 적극성: `적극 스위치 / 매치업 우선`

### C. `INTERIOR_ANCHOR`
1. 픽앤롤 커버리지: `드롭 / 헷지 / 앳더레벨 / 스위치`
   - 단, `Backline_Anchor`만 선택 가능
2. 림 보호 성향: `적극 블록 / 수직성 유지`
3. 스위치 허용 범위: `전 포지션 스위치 / 윙까지만 스위치 / 스위치 안함`
4. 미스매치 대응: `매치업 유지 / 스크램 스위치 시도`

## 4) "영향" 항목의 현행 매치엔진 반영 가능성 점검

아래는 **현재 코드에 이미 있는 연결점**을 기준으로 한 가능/불가 판단이다.

### 4-1) 이미 바로 연결 가능한 영향

1) 파울 확률
- 가능. `handle_foul`에서 `help_level`, `double_strength`, 접촉 멀티(`foul_contact_pmake_mult_*`) 등으로 파울 관련 확률이 이미 조정된다.
- 따라서 `핸들러 압박 강도`, `스틸 시도 성향`, `림 보호 성향`을 `help_level`/신규 압박 컨텍스트에 연결하면 즉시 반영 가능.

2) 턴오버 유발 확률 / 스틸 확률
- 가능. `handle_turnover`에서 `p_steal`, `steal_logit_delta`가 `d_press`, `help_level`, `double_strength`에 의해 조정된다.
- `스틸 시도 성향`, `핸들러 압박 강도`를 `d_press` 계수에 반영하면 구현 난이도 낮음.

3) 수비 리바운드 획득 확률
- 가능. `choose_drb_rebounder`에 `DRB_SOFTMAX_BETA`, `DRB_SOFTMAX_MIX`가 이미 있음.
- `리바운드 참여도`를 팀/개인 가중치로 매핑하기 좋음.

4) 체력(공격자+수비자)
- 부분 가능. 엔진은 시간/행동 기반 체력 소모 구조가 있고(`apply_time_cost` 경유), 선수 energy를 활용한다.
- 다만 "압박 강도별 추가 소모"는 별도 컨텍스트 계수(예: `DEF_PRESS_FATIGUE_MULT`)를 추가해야 함.

5) 상대 슛 성공 확률, 블록 확률
- 가능. 슛/파울 해결 로직이 `help_level`, 수비 quality delta에 의해 흔들림.
- 블록 확률은 전용 키가 약하므로 `rim contest` 계수(신규)로 SHOT_RIM 계열 priors/성공확률을 조정하는 방식이 현실적.

### 4-2) 현재 직접 연결이 약한(추가 구현 필요) 영향

1) 압박 지점 → 상대 샷클락 소모
- 현재는 possession 시간 소모가 액션 기반이며, 수비 압박 지점별 "초기 지연" 모델은 없음.
- 제안: possession 시작 시 `pos_start`와 연계해 `pre_action_delay_sec`를 주입.
  - 풀코트: +1.5~2.5초
  - 하프라인: +0.7~1.3초
  - 세트된 후: +0.0~0.4초

2) 돌파 허용 확률
- 전용 파라미터 없음.
- 제안: `Drive` 액션 priors에서 `SHOT_RIM_*`/`FOUL_DRAW_RIM`/`TO_HANDLE_LOSS`의 상대 가중치를 `POA_CONTAIN_LEVEL`로 조정.

3) 스크램 스위치 시도(미스매치 대응)
- 현재 matchups는 force/lock 개념은 있으나, "미스매치 탐지→자동 스크램" 룰은 약함.
- 제안: `make_possession_tactics_ctx` 단계에 `SCRAM_RULES`를 넣어 mismatch가 특정 임계 초과 시 `matchup_force(ttl=1)` 발행.

## 5) 선택 사항 별 게임 영향

본 섹션은 **미기재 항목 보완**이 아니라, 세 역할군의 **모든 선택 항목**에 대해 선택값별 게임 영향을 정리한 기준표다.
(잠금 역할은 서버 보정 이후 고정값이 그대로 적용됨)

### 5-1) `POA_DEFENDER` 선택 항목

#### 1) 스크린 대처 방식 (`파이트 스루 / 언더 / 스위치 / ICE`)
- **파이트 스루**
  - 영향: 볼핸들러 Pull-up 3/미드 효율 감소, 대신 수비 체력 소모 증가.
  - 엔진 반영: `SHOT_3_OD`, `SHOT_MID_PU` 성공/빈도 소폭 하향 + 수비 energy 소모 계수 상승.
- **언더**
  - 영향: 돌파 억제/림 허용 감소, 대신 Pull-up 3 허용 증가.
  - 엔진 반영: `SHOT_RIM_*`, `FOUL_DRAW_RIM` 하향 / `SHOT_3_OD` 상향.
- **스위치**
  - 영향: 1차 액션 저지는 쉬우나 미스매치 노출 증가.
  - 엔진 반영: 스크린 직후 `matchup_force` 발행 확률 상승, mismatch penalty(포스트/아이솔) 가중.
- **ICE**
  - 영향: 사이드 PnR에서 가운데 진입 억제, 리셋/사이드 유도 증가.
  - 엔진 반영: `Drive`의 중앙 진입 성공률 하향, `RESET_*`/패스 아웃컴 소폭 상향.

#### 2) 핸들러 압박 강도 (`강하게 / 보통 / 약하게`)
- **강하게**: 턴오버/스틸 유발↑, 파울↑, 양 팀 가드 체력 소모↑.
- **보통**: 기준값(밸런스).
- **약하게**: 파울/체력 소모↓, 대신 핸들러 의사결정 여유↑.
- 엔진 반영: `d_press`, `steal_logit_delta`, foul 관련 로짓, possession 내 에너지 소모 계수.

#### 3) 스틸 시도 성향 (`적극적 / 균형 / 적게 시도`)
- **적극적**: `STL%`, live-ball TO 유도↑, reach-in/도박수 실패로 돌파 허용↑.
- **균형**: 기준값.
- **적게 시도**: 스틸/턴오버 유도↓, 수비 위치 유지로 파울/직선 돌파 리스크↓.
- 엔진 반영: `p_steal`, `TO_BAD_PASS/TO_HANDLE_LOSS` split, 실패 시 `SHOT_RIM_*` 보정.

#### 4) 압박 지점 (`풀코트 / 하프라인 / 세트된 후`)
- **풀코트**: 상대 샷클락 소모↑, 초반 공격 전개 지연, 체력 소모↑.
- **하프라인**: 중간 수준 지연/체력 소모.
- **세트된 후**: 지연 효과 최소, 하프코트 수비 집중.
- 엔진 반영: possession 시작 pre-action delay(초), 양팀 ball-handler/온볼 수비 체력 소모 계수.

### 5-2) `WING_DEFENDER` 선택 항목

#### 1) 헬프 강도 (`강하게 / 보통 / 약하게`)
- **강하게**: 림/포스트 실점 억제↑, 반대편 킥아웃 3 허용↑.
- **보통**: 기준값.
- **약하게**: 킥아웃 3 억제, 대신 림 보호 부담을 인테리어에 전가.
- 엔진 반영: `help_level`(이미 존재), `SHOT_RIM_*` vs `SHOT_3_CS` 트레이드오프.

#### 2) 리바운드 참여도 (`적극 참여 / 보통 / 백코트 우선`)
- **적극 참여**: 수비 리바운드 획득↑, 세컨찬스 허용↓, 체력 소모↑.
- **보통**: 기준값.
- **백코트 우선**: 수비 전환 안정성↑, 대신 DRB 경합 참여↓.
- 엔진 반영: `choose_drb_rebounder` 가중치(`DRB_SOFTMAX_*`/개인 multiplier), 전환 시작 포지션 가중.

#### 3) 스위치 적극성 (`적극 스위치 / 매치업 우선`)
- **적극 스위치**: 1차 오픈 억제↑, 커뮤니케이션 실패/미스매치 리스크↑.
- **매치업 우선**: 미스매치 억제↑, 대신 스크린 대응 지연 시 오픈 허용 가능.
- 엔진 반영: `matchup_force`, `matchups_temp_locks` 발행 빈도, switch 허용 임계치.

### 5-3) `INTERIOR_ANCHOR` 선택 항목

#### 1) 픽앤롤 커버리지 (`드롭 / 헷지 / 앳더레벨 / 스위치`)
- **드롭**: 림 보호/롤맨 억제↑, 풀업 점퍼 허용↑.
- **헷지**: 볼 압박↑, 롤 복귀 지연 시 숏롤 허용↑.
- **앳더레벨**: 볼핸들러 1차 저지↑, 백라인 로테이션 부담↑.
- **스위치**: 즉시 액션 차단↑, 빅-가드 미스매치 노출↑.
- 엔진 반영: (가변 역할에서) `PnR/PnP` 관련 action/outcome priors 및 matchups 교체 정책.

#### 2) 림 보호 성향 (`적극 블록 / 수직성 유지`)
- **적극 블록**: 블록/림 슛 저지↑, 슈팅 파울↑.
- **수직성 유지**: 파울↓, 블록은 줄지만 컨테스트 안정성↑.
- 엔진 반영: `SHOT_RIM_*` 성공률, 블록 이벤트 확률, `FOUL_DRAW_RIM` 보정.

#### 3) 스위치 허용 범위 (`전 포지션 스위치 / 윙까지만 스위치 / 스위치 안함`)
- **전 포지션 스위치**: 2인 게임 즉시 대응↑, 미스매치 노출↑.
- **윙까지만 스위치**: 가드-윙 대응 유연성 확보, 빅 보호 유지.
- **스위치 안함**: 매치업 유지/림 보호 안정, 대신 스크린 추격 부담↑.
- 엔진 반영: switch resolver 필터(`all / 1to4 / none`)와 mismatch penalty 강도.

#### 4) 미스매치 대응 (`매치업 유지 / 스크램 스위치 시도`)
- **매치업 유지**: 로테이션 단순/안정, 장기 미스매치 노출 가능.
- **스크램 스위치 시도**: 불리 매치업 해소↑, 로테이션 실패 시 오픈 3 리스크↑.
- 엔진 반영: mismatch 탐지 임계치 초과 시 `matchup_force(ttl=1)` 또는 `double_active` 보조 발행.

## 7) 구현 우선순위(추천)

1. **1차(즉시 효과)**: `핸들러 압박 강도`, `스틸 시도 성향`, `헬프 강도`, `리바운드 참여도`, `림 보호 성향`
   - 기존 `help_level`, `d_press`, `choose_drb_rebounder`, shot/foul resolver에 붙이기 쉬움.
2. **2차(중간 난이도)**: `압박 지점`, `스위치 적극성`, `스위치 허용 범위`
   - matchups + sim_clock 연계 필요.
3. **3차(고난이도)**: `스크린 대처 방식` 상세(ICE/언더/파이트스루), `스크램 스위치`
   - 이벤트 레벨 스크린/미스매치 탐지 강화가 선행되어야 함.

이 순서로 가면 "영향이 체감되는 옵션"부터 빠르게 제공하고, 이후 전술 정밀도를 높일 수 있다.
