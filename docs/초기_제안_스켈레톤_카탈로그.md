# 초기 제안(Initial Offer) 스켈레톤 카탈로그

본 문서는 현재 코드 기준으로, 트레이드 생성기의 **초기 제안 단계**에서 생성 가능한 스켈레톤(제안 형태)을 정리한다.

## 1) 초기 제안이 스켈레톤으로 만들어지는 방식

- BUY/SELL 모두 `build_offer_skeletons_*`에서 registry(`build_default_registry`)를 사용한다.
- focal asset(타깃 선수/매물)에 대해 `classify_target_tier()`로 tier를 정하고, tier별 route에 포함된 `skeleton_id`만 활성화한다.
- 생성된 후보는 modifier(픽 보호 강도 변경, first↔swap 대체, second 재균형)로 변형될 수 있고,
  마지막으로 shape 제한(`max_assets_per_side`, `max_players_per_side`, `max_players_moved_total`, `max_picks_per_side`)을 통과해야 남는다.

## 2) target tier 분류

- `PICK_ONLY`: need_tag/match_tag에 PICK이 포함
- `ROLE`: 저가 만기자산 등 (expiring & market<=58 & salary<=30), 또는 market<52
- `STARTER`: market>=52
- `HIGH_STARTER`: market>=72
- `STAR`: market>=86

## 3) 현재 기본 스켈레톤 총계

- registry 기본 스펙: **27개**
- 도메인별
  - compat: 8
  - player_swap: 8
  - timeline: 3
  - salary_cleanup: 4
  - pick_engineering: 4

## 9) 스켈레톤 27종 상세 설명 (의도/언제/주고받는 자산)

아래 설명은 각 빌더 구현의 실제 자산 구성 로직을 사람이 이해하기 쉽게 풀어쓴 것이다.

### A. compat (8종)

1. `compat.picks_only` (BUY)
- 의도: 선수를 보내지 않고 cap space로 상대 핵심 타깃을 사오는 **순수 픽 매입**.
- 언제: 구매팀이 타깃 연봉을 흡수할 수 있을 때(`_can_absorb_without_outgoing`).
- 주고받는 자산:
  - seller → buyer: 타깃 선수 1명
  - buyer → seller: 픽 패키지(기본 SECOND 우선, 필요 시 FIRST_SAFE 포함; 보통 1장, seller가 REBUILD면 최대 2장)

2. `compat.young_plus_pick` (BUY)
- 의도: 즉시전력 대신 **젊은 자원 + 소량 픽**으로 타깃을 사오는 형태.
- 언제: seller 타임라인이 REBUILD/RE_TOOL이면 유망주 우선, 아니면 저비용 young throw-in 위주.
- 주고받는 자산:
  - seller → buyer: 타깃 선수 1명
  - buyer → seller: 젊은 선수 1명 + SECOND 계열 픽 최대 1장

3. `compat.p4p_salary` (BUY)
- 의도: **연봉대 맞교환(1-for-1)** 중심의 가장 기본적인 선수 교환형.
- 언제: seller가 WIN_NOW/RE_TOOL 성향일 때 잘 맞음(즉시 활용 가능한 리턴 필요).
- 주고받는 자산:
  - seller → buyer: 타깃 선수 1명
  - buyer → seller: 연봉/니즈 유사 선수 1명

4. `compat.consolidate_2_for_1` (BUY)
- 의도: 구매팀이 선수 2명을 묶어 보내고 상위 1명을 받는 **로스터 압축(consolidation)**.
- 언제: 상위 선수 업그레이드가 필요하고 buyer가 CONSOLIDATE+FILLER_CHEAP 조합을 보유할 때.
- 주고받는 자산:
  - seller → buyer: 타깃 선수 1명
  - buyer → seller: 선수 2명(CONSOLIDATE 1 + FILLER_CHEAP 1) + SECOND 계열 픽 최대 1장

5. `compat.buyer_picks` (SELL)
- 의도: 판매팀 관점에서 매물을 **픽 수익화**.
- 언제: buyer가 매물 연봉을 흡수 가능할 때.
- 주고받는 자산:
  - seller → buyer: sale_asset 선수 1명
  - buyer → seller: 픽 패키지(SECOND 우선, 필요 시 FIRST_SAFE; REBUILD 상대면 최대 2장)

6. `compat.buyer_young_plus_pick` (SELL)
- 의도: 판매팀이 매물을 보내고 **젊은 자산 + 픽** 회수.
- 언제: seller가 리툴/리빌드일 때 특히 자연스러운 구조.
- 주고받는 자산:
  - seller → buyer: sale_asset 선수 1명
  - buyer → seller: 젊은 선수 1명 + SECOND 계열 픽 최대 1장

7. `compat.buyer_p4p` (SELL)
- 의도: 판매팀이 전력 공백을 줄이기 위해 **즉시전력 1명**을 되돌려 받는 매각.
- 언제: seller가 WIN_NOW/RE_TOOL일 때만 활성.
- 주고받는 자산:
  - seller → buyer: sale_asset 선수 1명
  - buyer → seller: 연봉대 유사 선수 1명

8. `compat.buyer_consolidate` (SELL)
- 의도: 판매팀이 1명을 보내고 **2명+픽**으로 로테이션 폭을 넓힘.
- 언제: buyer에 CONSOLIDATE/FILLER_CHEAP 자원이 있을 때.
- 주고받는 자산:
  - seller → buyer: sale_asset 선수 1명
  - buyer → seller: 선수 2명 + SECOND 계열 픽 최대 1장

### B. player_swap (8종)

9. `player_swap.role_swap_small_delta`
- 의도: 큰 구조 변경 없이 **소규모 가치 차(delta)를 픽으로 보정**하는 맞교환.
- 언제: ROLE/STARTER급에서 과도한 딜 대신 미세 조정이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 연봉대 유사 선수 1명 + (SECOND 또는 SWAP) 최대 1장

10. `player_swap.fit_swap_2_for_2`
- 의도: 상대팀 니즈 맞춤으로 2명을 보내는 **핏 보완형 다자 교환**.
- 언제: 단일 선수로는 상대 fit이 안 맞고, 조합으로 맞춰야 할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 선수 2명(CONSOLIDATE 1 + FILLER_CHEAP 1)

11. `player_swap.starter_for_two_rotation`
- 의도: 상위 스타터 1명 ↔ 회전 자원 2명 성격의 딜(코드상 fit_swap_2_for_2 기반 재태깅).
- 언제: 스타터급 자산의 depth 분산 트레이드가 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 선수 2명(실제 구성은 fit_swap_2_for_2와 동일 베이스)

12. `player_swap.one_for_two_depth`
- 의도: `starter_for_two_rotation`의 저강도 버전. **1명 받고 depth 2명 보내기**.
- 언제: ROLE/STARTER/HIGH_STARTER에서 로스터 깊이 조정이 목적일 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: FILLER_CHEAP 1명 + 연봉완화형 선수 1명(대상 연봉의 약 45% 기준)

13. `player_swap.three_for_one_upgrade`
- 의도: 고가 타깃 확보를 위한 **3-for-1 업그레이드 패키지**.
- 언제: 상위 티어(STAR/HIGH_STARTER)로 점프할 때 자주 필요한 구조.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 선수 3명(CONSOLIDATE 1 + FILLER_CHEAP 1 + 추가 salary-ish 1)

14. `player_swap.bench_bundle_for_role`
- 의도: 벤치 묶음(2명) + 픽으로 ROLE급 거래를 성사시키는 패키지.
- 언제: 상위 자산까지는 아니지만, depth 재배치/벤치 압축이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 선수 2명(fit_swap 베이스) + SECOND 최대 2장

15. `player_swap.change_of_scenery_young`
- 의도: 커리어 전환/환경 변화가 필요한 젊은 선수끼리의 **change-of-scenery**.
- 언제: 팀 전술/역할 불일치로 유망주의 리셋이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 젊은 선수 1명( prospect 우선, 없으면 throw-in )

16. `player_swap.star_lateral_plus_delta`
- 의도: 고티어 자산을 **동급 맞교환 + 소폭 보정(delta)**으로 처리.
- 언제: STAR/HIGH_STARTER 라우트에서 “큰 방향은 lateral, 차이는 부가자산으로 보정”이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 연봉대 유사 선수 1명 + 소규모 부가자산(베이스는 role_swap_small_delta)

### C. timeline (3종)

17. `timeline.veteran_for_young`
- 의도: 즉시전력(베테랑)과 미래가치(젊은 선수)를 맞바꾸는 **타임라인 스왑**.
- 언제: 한 팀은 당장 성적, 다른 팀은 미래 성장에 무게를 둘 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 젊은 선수 1명

18. `timeline.veteran_for_young_plus_protected_first`
- 의도: 타임라인 스왑에 **보호된 1라 성격의 픽**을 추가해 가치 균형.
- 언제: 단순 1:1 young 교환으로는 가치차가 클 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 젊은 선수 1명 + FIRST_SAFE/SECOND 계열 픽 최대 1장

19. `timeline.bluechip_plus_first_plus_swap`
- 의도: 블루칩급 젊은 자원 + 1라 + 스왑까지 포함한 **하이엔드 미래 패키지**.
- 언제: STAR/HIGH_STARTER급 확보처럼 가격이 높은 거래에서.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 젊은 선수 1명 + (FIRST_SAFE, SWAP, SECOND 중) 최대 2장

### D. salary_cleanup (4종)

20. `salary_cleanup.rental_expiring_plus_second`
- 의도: 만기/단기 자산 정리 성격의 거래에 **2라운드 보정**을 붙인 형태.
- 언제: 연봉 맞춤 선수 1명을 함께 보내며 부담을 줄이고 싶을 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: salary-ish 선수 1명 + SECOND 최대 1장

21. `salary_cleanup.pure_absorb_for_asset`
- 의도: 구매팀 cap space로 계약을 흡수하고, seller는 자산(픽)을 받는 **순수 흡수 딜**.
- 언제: buyer가 연봉을 outbound 없이 감당 가능할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 픽(SECOND/FIRST_SAFE 우선) 최대 2장

22. `salary_cleanup.partial_dump_for_expiring`
- 의도: 전액이 아니라 일부 연봉만 맞춰 보내는 **부분 덤프**.
- 언제: 완전 흡수는 어렵지만, 부담을 줄여 이동시키려는 경우.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 연봉 약 70% 타깃의 filler 1명 + SECOND 최대 1장

23. `salary_cleanup.bad_money_swap`
- 의도: 비선호 계약끼리의 **bad money 교환**에 픽을 소량 덧붙여 성사율 확보.
- 언제: 양측 모두 계약 구조 개선이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: 연봉대 유사 선수 1명 + SECOND 최대 1장

### E. pick_engineering (4종)

24. `pick_engineering.first_split`
- 의도: 1라 가치를 여러 조각으로 설계해 전달하는 **픽 분할형**.
- 언제: 단일 고가픽 대신 분산된 픽 패키지 제안이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: FIRST_SAFE + SECOND 우선 조합, 최대 2장

25. `pick_engineering.second_ladder_to_protected_first`
- 의도: 다수 2라를 쌓아 보호된 1라 급 가치로 끌어올리는 **ladder 구조**.
- 언제: 확정 1라를 꺼리지만 총가치는 맞춰야 할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: SECOND, SECOND, FIRST_SAFE 우선 조합, 최대 3장

26. `pick_engineering.swap_purchase`
- 의도: 픽 스왑권을 매개로 가치 이전하는 **swap 구매형**.
- 언제: 직접 1라 지출 없이 유사한 미래가치 이전이 필요할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: SWAP 우선 + SECOND 보조, 최대 2장

27. `pick_engineering.swap_substitute_for_first`
- 의도: 1라를 스왑들로 대체해 리스크를 분산하는 **first 대체형**.
- 언제: 1라 자체를 보내기 어려운 팀이 대안을 제시할 때.
- 주고받는 자산:
  - seller → buyer: focal 선수 1명
  - buyer → seller: SWAP, SWAP, SECOND 우선 조합, 최대 3장
