## 2) 이슈 1 검증: 픽 기대값 → 분포/시나리오 자산

## 2-1. 구현된 점 (긍정)

1. **분포형 타입/엔진이 실제로 도입됨**
   - `PickDistributionBundle`(pmf/cdf/ev/variance/tail/scenario_notes/source_coverage) 정의 및 생성 로직 존재.
   - standings + lottery rules 기반 PMF 구성, 보호/스왑 반영 루틴, 확률 질량 정규화/검증이 구현됨.

2. **기존 provider 경로로 연결됨**
   - `data_context`에서 `build_pick_distributions_from_standings()` 호출 후,
     `pick_distributions` 저장 + `PickExpectation`을 `distribution_ev` 기반 호환 뷰로 생성.

3. **가격 모듈이 분포 입력을 읽도록 확장됨**
   - `market_pricing`에서 distribution 우선으로 `compat_expected_pick_number` 읽고,
     `variance`, `tail_upside_prob`, `tail_downside_prob` 기반 `PICK_DISTRIBUTION_ADJUST` 보정 단계 추가.

## 2-2. 미완료/위험

1. **실패 중인 테스트 1건 (즉시 수정 필요)**
   - `test_pick_distribution_is_primary_input_when_present` 실패.
   - 원인: `_pick_distribution_signature()`가 Mapping(dict)만 가정하고 `distribution.get`, `k in distribution`을 사용.
   - 실제 호출 값은 `PickDistributionBundle` dataclass 객체이므로 `TypeError` 발생.

2. **v1 helper가 파일에 잔존**
   - `build_pick_expectations_from_standings()`가 `data_context.py`에 여전히 존재하고 검색 기준 단독 정의(미사용).
   - 전환 문맥상 "삭제 대상 레거시" 후보.

3. **완전 v2 직결이 아닌 compat 중심 경로 유지**
   - 여전히 `compat_expected_pick_number`를 통해 v1 곡선(`_pick_number_bonus`)을 타는 구조.
   - 즉, "분포 직접 가격"으로 완전 전환된 상태는 아님(현재는 하이브리드).

## 2-3. 판단

- **구현 방향 자체는 맞고 핵심 구조는 반영됨**
- 단, **현재 브랜치 그대로 운영 전환하기엔 안정성(테스트 fail)과 레거시 정리도가 부족**

## 3) 이슈 2 검증: 단일 archetype·salary×years → 역할 질감 + 계약 질감

## 3-1. 구현된 점 (긍정)

1. **Role Texture 모듈 도입/연결 완료**
   - `RoleTexture` 타입과 `build_role_textures()` 구현.
   - `player.meta.role_fit` / `attrs.role_fit` 우선, `fit_engine` 공급벡터 보강, coverage/notes 기록.

2. **Contract Texture 모듈 도입/연결 완료**
   - `ContractTexture` 타입과 `build_contract_textures()` 구현.
   - 옵션 방향(TO/PO/ETO), trigger_risk, reversibility, matching_utility, toxic_risk 등 분리 지표 계산.

3. **package_effects가 texture 기반으로 전환됨**
   - 중복/체감 감소가 단일 archetype bucket이 아닌 `RoleTexture` 벡터 overlap 행렬 기반.
   - CAP_FLEX 커미트먼트가 단순 salary×years helper 대신 contract texture 기반 multiplier(제어권/트리거/독성 반영)로 계산.
   - `CONTRACT_COMPONENT` / `BASKETBALL_COMPONENT` 단계 분리 노출.

4. **관련 테스트 다수 통과**
   - texture component 테스트/ context_v2 helper 테스트 통과.

## 3-2. 미완료/정리 필요

1. **설정/주석의 레거시 흔적 정리 필요**
   - `cap_flex_scale` 주석이 여전히 `salary*years` 표현을 유지.
   - 실제 로직은 texture commitment 기반이므로 문구 불일치.

2. **dual-read 메타가 사실상 고정 0으로 남아 있음**
   - `v2_texture_diff.basketball_component_delta`가 현재 0 고정 구조.
   - dual-read 유지 목적이라면 v1-v2 실측 diff를 남기거나, 목적 종료 시 메타 단순화 필요.

3. **RoleTexture 축이 설계 대비 축약 버전**
   - 구현구상서의 상세 축(온볼/오프볼/스크린/POA/헬프/멀티포지션 등) 대비,
     현재는 `creation/spacing/rim_pressure/defense/connector` 중심.
   - "최소 실행 가능 버전"으로는 타당하나 설계서 1차 목표 대비는 축소 상태.

## 3-3. 판단

- **핵심 목표(단일 archetype 제거, salary×years 단독 의존 제거)는 달성됨**
- 다만 **설계서 풀스펙 대비로는 R1 범위 구현 + 레거시/설명 정합성 보강 필요**
