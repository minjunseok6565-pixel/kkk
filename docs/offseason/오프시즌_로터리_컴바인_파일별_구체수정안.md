# 오프시즌 로터리/컴바인 단계형 공개: 파일별 구체 수정안

## 문서 목적
- 이전 브리핑을 실제 구현 태스크로 바로 옮길 수 있도록, **파일별 변경 포인트**를 구체적으로 정리한다.
- 전제: **신규 API 신설 없이** 기존 API만 재사용한다.
- 수정 대상: `static/NBA.html`, `static/NBA.css`, `static/js/...` 프론트 파일만.

---

## 0) 최종 UX 단계(목표 상태)
`offseasonDevFlow`에서 은퇴 이후 단계를 다음처럼 분해한다.

1. `LOTTERY_ODDS` : 기존 로터리 확률(1~14) 노출
2. `LOTTERY_IN4_REVEALED` : 당첨팀(in 4)만 공개
3. `LOTTERY_TOP4_ORDER_REVEALED` : 1~4픽 순서 공개
4. `DRAFT_SETTLED_ROUND1` : settle 후 1~30픽(1라운드 전체) 공개
5. `COMBINE_OVERVIEW` : 컴바인 항목별 Top10 카드
6. `COMBINE_DETAIL` : 카드 클릭 시 해당 항목 전체 리스트

> 핵심: `POST /api/offseason/draft/lottery` 결과를 프론트에서 **2단계(in4 → 1~4)** 로 나눠 보여주고,
> `POST /api/offseason/draft/settle`는 1~4 공개 다음 단계에서만 호출한다.

## 1) 파일별 수정안

## 1-1) `static/js/app/state.js`
### 변경 목적
오프시즌 단계를 세분화하고, 컴바인 화면에 필요한 프론트 상태를 저장한다.

### 수정 항목
- `offseasonDev.step` 주석/허용값 확장
  - 기존: `... | RETIREMENT_PROCESSED | LOTTERY_SETTLED`
  - 변경: `... | RETIREMENT_PROCESSED | LOTTERY_ODDS | LOTTERY_IN4_REVEALED | LOTTERY_TOP4_ORDER_REVEALED | DRAFT_SETTLED_ROUND1 | COMBINE_OVERVIEW | COMBINE_DETAIL`
- `offseasonDev` 내부 필드 추가
  - `draftCombineResult: null` (combine 실행 결과)
  - `draftBundleResult: null` (bundle 조회 원본)
  - `combineCategoryCards: []` (카드 렌더용 Top10 집계)
  - `combineSelectedCategory: ""` (상세로 들어간 항목 key)
  - `combineDetailRows: []` (선택 항목 전체 정렬 rows)
  - `busy: false` (다음 버튼 연타 방지용 플래그; `loading`과 별도)

## 1-2) `static/js/features/offseason/offseasonDevFlow.js`
### 변경 목적
현재 단일 `LOTTERY_SETTLED` 단계를 로터리/정산/컴바인 단계로 분해하고 렌더링을 추가한다.

### A. 상태/헬퍼 추가
- `ensureOffseasonDevState()` 기본값에 state.js와 동일한 신규 필드 반영.
- 로터리 파싱 헬퍼
  - `getLotteryResult()` : `draftLotteryResult.plan.lottery_result` 안전 접근
  - `getWinnersTop4()` : `winners_top4` 배열 normalize
- 컴바인 집계 헬퍼
  - `extractCombineRowsFromBundle(bundle)` : `bundle.pool.prospects[*].meta.combine` 추출
  - `buildTop10ByCategory(rows, categoryKey)`
  - `buildCombineCategoryCards(rows, categoryKeys)`
  - `buildCombineDetailRows(rows, selectedCategory)`

### B. 렌더 함수 분리
기존 `renderOffseasonDevFlow()` 내 분기 확장:

1) `LOTTERY_ODDS`
- 기존 `renderLotteryOddsRows()` 재사용
- Next 라벨: `다음으로 (로터리 in 4 공개)`

2) `LOTTERY_IN4_REVEALED`
- 신규 `renderLotteryIn4Rows()`
  - `winners_top4` 팀 4개를 “당첨” 카드로만 표시(순번 미노출)
- Next 라벨: `다음으로 (1~4픽 공개)`

3) `LOTTERY_TOP4_ORDER_REVEALED`
- 신규 `renderLotteryTop4OrderRows()`
  - 같은 `winners_top4`를 슬롯 1~4로 표시
- Next 라벨: `다음으로 (보호/스왑 반영 후 1~30픽)`

4) `DRAFT_SETTLED_ROUND1`
- 기존 `renderOwnershipRows()`를 대체/확장해서 `slot 1..30` 표시
- 신규 `renderRound1FinalOrderRows()`
  - 소팅: `slot ASC`
  - 열: `픽`, `pick_id`, `원 소유`, `최종 지명권 팀`
- Next 라벨: `다음으로 (컴바인)`

5) `COMBINE_OVERVIEW`
- 신규 `renderCombineOverviewCards()`
  - 항목별 Top10 미니카드(예: `vertical`, `lane_agility`, `sprint_3_4`, `shuttle`, `standing_reach`)
  - 카드 클릭 버튼에 `data-offseason-combine-category="..."`
- Next 라벨: `완료` 또는 `다음으로 (메인)` 정책 선택

6) `COMBINE_DETAIL`
- 신규 `renderCombineDetailTable()`
  - 선택 항목 기준 전체 대상 정렬 테이블
- 보조 액션 버튼(목록 복귀) 제공

### C. 단계 전이 로직(`advanceOffseasonDevStep`) 수정
권장 전이:
- `RETIREMENT_PROCESSED` → **lottery만 호출** 후 `LOTTERY_ODDS`
- `LOTTERY_ODDS` → `LOTTERY_IN4_REVEALED` (API 호출 없음)
- `LOTTERY_IN4_REVEALED` → `LOTTERY_TOP4_ORDER_REVEALED` (API 호출 없음)
- `LOTTERY_TOP4_ORDER_REVEALED` → settle 호출 후 `DRAFT_SETTLED_ROUND1`
- `DRAFT_SETTLED_ROUND1` → combine + bundle 호출 후 `COMBINE_OVERVIEW`

신규 함수 제안:
- `runLotteryStepOnly()`
- `runSettleStepOnly()`
- `runCombineAndLoadBundleStep()`

### D. 이벤트 처리 확장(위임)
`offseasonDevContent` 클릭 위임에 추가:
- `button[data-offseason-combine-category]` 클릭 시
  - `combineSelectedCategory` 세팅
  - `combineDetailRows` 계산
  - `step = "COMBINE_DETAIL"`
  - `renderOffseasonDevFlow()`
- `button[data-offseason-combine-back]` 클릭 시
  - `step = "COMBINE_OVERVIEW"`

### E. 안정성
- `advanceOffseasonDevStep` 시작 시 `if (flow.busy) return;` 가드
- `try/finally`에서 `busy` true/false 관리
- 실패 시 동일 step 유지 + status에 재시도 안내

## 2) 기존 API 재사용 매핑 (신규 API 없음)

### 로터리/정산
- `POST /api/offseason/draft/lottery`
  - 사용 시점: `RETIREMENT_PROCESSED -> LOTTERY_ODDS`
  - 재사용 데이터: `plan.lottery_result.winners_top4` (in4/1~4 모두 동일 데이터)

- `POST /api/offseason/draft/settle`
  - 사용 시점: `LOTTERY_TOP4_ORDER_REVEALED -> DRAFT_SETTLED_ROUND1`
  - 재사용 데이터: `turns` (round=1, slot 1..30)

### 컴바인
- `POST /api/offseason/draft/combine`
  - 사용 시점: `DRAFT_SETTLED_ROUND1 -> COMBINE_OVERVIEW` 진입 직전
- `GET /api/offseason/draft/bundle`
  - 사용 시점: combine 이후 즉시 호출
  - 재사용 데이터: `pool.prospects[*].meta.combine`
