# 오프시즌 DEV 임의 진행 프론트 구현 작업 계획

## 목표
- 이번 문서는 **프론트 구현 중심**으로,
  - DEV용 `임의 진행` 버튼 추가
  - 플레이오프 우승 화면(임시)
  - 오프시즌 단계별 화면/상태 전환
  - 신규 백엔드 API(`contracts/expired`, `options/player/results`) 연동
  까지를 실제 파일 단위로 쪼개서 계획한다.

## 1) HTML 작업 계획 (`static/NBA.html`)

## 1-1. 메인 화면 DEV 액션 버튼 확장
- 위치: 기존 `next-game-actions` 블록 하단(현재 `10경기 진행 (DEV)` 버튼 옆)
- 추가 요소:
  - `id="next-game-dev-offseason-btn"` 버튼
  - 라벨: `오프시즌 임의 진행 (DEV)`

## 1-2. 신규 스크린 섹션 추가
기존 스크린 구조(`section#...-screen.panel`)를 따라 아래 스크린을 추가한다.

1. `#offseason-dev-champion-screen`
   - 목적: 플레이오프 우승 임시 화면
   - 주요 요소:
     - 우승팀/시즌 요약 텍스트
     - `id="offseason-dev-enter-btn"` (`오프시즌으로`)

2. `#offseason-dev-flow-screen`
   - 목적: 오프시즌 단계 진행 메인 컨테이너(1개 스크린 + 내부 단계 렌더링)
   - 주요 요소:
     - 단계 타이틀/설명 영역
     - 단계별 리스트 렌더링 영역 (`id="offseason-dev-content"`)
     - 공통 상태/에러 영역 (`id="offseason-dev-status"`)
     - 하단 CTA 버튼
       - `id="offseason-dev-next-btn"` (`다음으로`)
       - 필요 시 단계 전용 보조 버튼 영역 (`id="offseason-dev-actions"`)

## 1-3. 단계별 렌더링을 위한 최소 placeholder
- `offseason-dev-content` 내부에 JS가 동적으로 테이블/카드를 그릴 수 있도록 빈 컨테이너만 두고,
  HTML에는 정적 테이블 마크업을 과도하게 넣지 않는다.
- 이유: 단계별 데이터 구조가 크게 다름(대학 결과/선언자/옵션/만료계약/은퇴/로터리).

## 3-7. API 연동 상세

### A) 오프시즌 진입
- `POST /api/season/enter-offseason`
- 성공 시 step = `INTRO`

### B) 대학 finalize 및 결과
- `POST /api/offseason/college/finalize`
- 조회:
  - `GET /api/college/teams?season_year={from_year}`
  - `GET /api/college/players?draft_year={to_year}&declared_only=true`

### C) TEAM 옵션
- 조회: `POST /api/offseason/options/team/pending` (body: `user_team_id`)
- 결정: `POST /api/offseason/options/team/decide`
- 완료 후 반드시 `POST /api/offseason/contracts/process` 실행

### D) PLAYER 옵션 결과 (신규 API)
- `POST /api/offseason/options/player/results` (body: `team_id`, `include_pending`)
- 화면 표기:
  - `option_status`
  - `expected_effect`
  - `is_currently_fa`

### E) 만료 계약자 (신규 API)
- `POST /api/offseason/contracts/expired` (body: `team_id`)
- 행 액션:
  - `재계약 제의` → 계약 협상 플로우 진입 (`/api/contracts/negotiation/start` ...)
  - `방출` → `/api/contracts/release-to-fa`

### F) 은퇴
- 처리: `POST /api/offseason/retirement/process`
- 결과 목록 렌더

### G) 로터리/픽 소유권
- `POST /api/offseason/draft/lottery`
- `POST /api/offseason/draft/settle`
- settle 결과의 turn/ownership을 표로 표시

## 4) 구현 순서 (프론트 중심)

1. **구조 작업**: HTML 스크린 추가 + dom/router wiring
2. **진입 작업**: DEV 버튼 → 챔피언 화면 진입
3. **상태머신 골격**: `offseasonDevFlow.js` + next 버튼 단계 전환
4. **핵심 단계 연결**:
   - enter-offseason
   - college/finalize
   - team options + contracts/process
5. **신규 API 연동 단계**:
   - player option results
   - expired contracts
6. **후속 단계**:
   - retirement
   - lottery/settle
7. **마감 작업**:
   - 오류/로딩 UX 정리
   - 버튼 disabled/중복 클릭 방지
