# 개인 훈련 탭 프론트 구현 구상 (2026-03-23)

## 목표

훈련 화면에서 **개인 훈련 탭 진입**이 가능해야 하며,
다음 기능을 제공한다.

1. 팀 소속 선수 명단 표시
2. 선수 선택
3. 선택 선수의 개인 훈련 항목(primary/secondary) 설정
4. 훈련 강도(intensity) 설정
5. 저장 및 즉시 반영(캐시/상태 동기화)

---

## 현재 상태 요약

- 훈련 상단에 `개인 훈련 (준비중)` 버튼이 있지만 `disabled` 상태다.
- 현재 훈련 화면은 팀 훈련(`practice`) 중심으로 구성되어 있다.
- 개인 훈련 API는 이미 백엔드에 존재한다.
  - 조회: `GET /api/training/player/{player_id}`
  - 저장: `POST /api/training/player/set`

즉, 이번 작업의 핵심은 **프론트 UI + 이벤트 + 상태 + API 연결**이다.

---

## 백엔드 계약(프론트가 의존할 API)

### 1) 조회

- `GET /api/training/player/{player_id}?season_year={optional}`
- 응답 예시

```json
{
  "player_id": "1610612737_001",
  "season_year": 2026,
  "plan": {
    "primary": "SHOOTING",
    "secondary": "PLAYMAKING",
    "intensity": "MED"
  },
  "is_default": false
}
```

### 2) 저장

- `POST /api/training/player/set`
- 요청 바디

```json
{
  "player_id": "1610612737_001",
  "season_year": 2026,
  "primary": "SHOOTING",
  "secondary": "PLAYMAKING",
  "intensity": "HIGH"
}
```

- 성공 응답

```json
{ "ok": true, "player_id": "...", "season_year": 2026 }
```

## 4) `static/js/features/training/` 하위 신규 모듈

신규 파일 제안: `playerTrainingTab.js`

### 책임 분리

1. `loadPlayerTrainingRoster()`
   - 기존 `trainingRoster`(team-detail 결과) 재사용
   - 없다면 `prefetchTrainingCoreData`/`loadTrainingData` 이후 사용

2. `renderPlayerTrainingList()`
   - 로스터를 포지션/ovr 순 등으로 렌더
   - 선택 선수 하이라이트

3. `fetchPlayerTrainingPlan(playerId)`
   - `GET /api/training/player/{player_id}` 호출
   - `state.playerTrainingPlansByPlayerId[pid]` 캐시

4. `renderPlayerTrainingDetail(playerId)`
   - 선수 카드 + 현재 플랜 표시
   - primary/secondary/intensity 폼 렌더

5. `setPlayerTrainingDraftFromPlan(plan)`
   - API 응답 plan -> 드래프트 반영

6. `savePlayerTrainingPlan()`
   - `POST /api/training/player/set` 호출
   - 성공 시 캐시/상태/토스트 갱신

7. `activateTrainingTab(mode)`
   - team/player 탭 전환 및 패널 표시 제어

### 카테고리/강도 옵션 상수

```js
const PLAYER_TRAINING_CATEGORIES = [
  "BALANCED", "SHOOTING", "FINISHING", "PLAYMAKING",
  "DEFENSE", "REBOUNDING", "PHYSICAL", "IQ", "POST"
];

const PLAYER_TRAINING_INTENSITIES = ["LOW", "MED", "HIGH"];
```

> secondary에서 `BALANCED` 선택 시 `null`로 보정해 저장하는 UX를 권장.

## 7) `static/js/app/cachePolicy.js` (선택)

현재 팀 훈련 캐시 키는 있으나, 개인 훈련 플랜 캐시 키는 없다.

- `training:player-plan:{playerId}:{seasonYear}`

또는 단순화해서 초기에 `fetchJson`만 쓰고,
2차 작업으로 캐시 정책을 붙여도 된다.

## UI 동작 시나리오

1. 사용자가 훈련 화면 진입
2. 상단에서 `개인 훈련` 클릭
3. 좌측에 팀 로스터 노출
4. 선수 A 선택
5. API로 선수 A의 현재 plan 조회
6. 우측 폼에 `primary/secondary/intensity` 세팅
7. 값 변경 후 저장 클릭
8. 저장 성공 메시지 + 목록에 즉시 반영
9. 선수 B 선택 시 동일 플로우 반복

## 예외/에러 처리

1. `selectedTeamId` 없을 때
   - 기존과 동일하게 훈련 화면 진입 차단
2. 로스터가 비어 있을 때
   - "소속 선수가 없습니다" 안내
3. GET 실패
   - 기본 드래프트(`BALANCED/MED`)로 표시 + 경고 문구
4. POST 실패
   - draft 유지, 재시도 가능
5. 시즌 연도 누락
   - 백엔드가 state 시즌으로 fallback하므로 프론트는 season_year 생략 가능

## 간단 수용 기준(AC)

1. 훈련 화면에서 개인 탭 버튼이 활성화되어 클릭 가능
2. 개인 탭에서 현재 팀 로스터가 보임
3. 선수 선택 시 해당 선수의 개인 훈련 플랜이 로드됨
4. primary/secondary/intensity를 변경하고 저장 가능
5. 저장 성공 후 다시 조회 시 값이 유지됨
6. 오류 시 사용자에게 실패 메시지가 표시됨
