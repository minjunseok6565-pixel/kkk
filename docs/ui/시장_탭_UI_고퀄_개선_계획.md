# 시장 탭 UI 고퀄리티 개선 계획

## 1) 문제 정의
- 현재 `시장` 탭은 다크 네온 톤(강한 그라데이션 + 어두운 카드) 중심이고, `메디컬 센터`/`홈 대시보드`는 라이트 톤의 카드형 정보 구조입니다.
- 같은 앱 내 탭임에도 시각 언어(배경/테두리/버튼/타이포 위계)가 달라 사용자에게 “별도 제품”처럼 느껴집니다.
- 요구사항: **노출 정보와 기능은 그대로 유지**, 디자인만 상위 품질로 개선.

## 2) 목표 (Design Goals)
1. **일관성**: 홈/메디컬과 동일한 디자인 시스템(라이트 카드, 부드러운 보더, 여백 리듬)으로 통일.
2. **가독성**: 표/카드/액션 버튼의 시선 흐름을 명확하게 재정렬.
3. **고급감**: 미세한 깊이감(그림자/그라데이션/포커스 링)과 상태 표현(hover/active/disabled)을 정교화.
4. **무손실 개편**: 텍스트/지표/버튼/탭/테이블 컬럼은 변경 없이 UI 스킨만 개선.

## 3) 핵심 디자인 방향

### A. 화면 레벨 톤 정렬
- `시장` 탭의 전체 배경을 홈/메디컬과 같은 **밝은 중성 배경**으로 전환.
- 기존 강한 네온 그라데이션 배경 제거, 필요 시 상단 1~2px 포인트 라인만 사용.
- 제목/서브타이틀 색상은 홈과 같은 계열(`#0f172a`, `#64748b` 중심)로 맞춤.

### B. 상단 헤더 정보 구조 고급화
- 현재 헤더(`MARKET`, `시장`, 설명, 메인으로 버튼)를 **카드형 헤더 블록**으로 통합.
- 우측 CTA(메인으로)는 라운드 + 얇은 보더 + hover 하이라이트로 통일.
- 헤더 하단에 `FA / 트레이드 블록 / 제안·협상` 세그먼트 탭을 배치해 정보 구조를 1차 정리.

### C. 서브탭/필터 컨트롤 일관화
- `시장 하위 탭`, `트레이드 블록 하위 탭` 모두 동일한 탭 컴포넌트 룩으로 통일:
  - 기본: 화이트/라이트그레이 배경
  - 활성: 블루 틴트 + 선명한 텍스트
  - hover/focus-visible 상태 분리
- 버튼 높이, 패딩, 폰트 굵기, 반경을 홈/메디컬 버튼 규격과 동일하게 맞춤.

### D. 테이블/리스트 시인성 업그레이드
- `FA`, `트레이드 블록` 표에 다음 적용:
  - 헤더 행을 옅은 틴트 배경으로 고정
  - 행 간 구분선 명확화 + hover 시 하이라이트
  - 숫자 컬럼 가독성 향상을 위한 정렬 규칙 통일
- 빈 상태 메시지(예: 등록된 선수가 없습니다)는 카드 내부 **중앙 정렬 + 보조 설명 톤**으로 고급화.

### E. 제안/협상(Inbox) 카드 고품질 리스킨
- 기존 다크 카드 스타일을 라이트 카드 스타일로 전환하되 정보 배치는 유지.
- 카드 구조를 `좌측 로고/메타 – 중앙 패키지 2열 – 우측 액션`으로 유지하면서:
  - 카드 보더/그림자 약화(과한 대비 제거)
  - 패키지 박스 배경 톤 분리로 정보 군집 강화
  - 액션 버튼 우선순위(수락/거절/상세) 색상 체계 명확화

### F. 모달(트레이드 딜) 시각 품질 개선
- 모달 배경/테두리/헤더를 라이트 테마로 맞추고, 스크롤 영역 대비 개선.
- 상단 패키지 요약과 하단 자산 선택 영역의 시각적 위계를 강화.
- 선택 상태, hover, 비활성 상태를 명확히 분리해 조작 피드백 강화.

## 4) 구현 계획 (파일 단위)

### 4-1. `static/css/screens/market.css`
- 시장 화면 전반의 색상/배경/보더/그림자 체계를 라이트 톤으로 재정의.
- 탭, 표, 카드, 모달 관련 market 네임스페이스 스타일을 전면 리파인.
- 상태 스타일(hover, active, focus-visible, disabled) 정교화.

### 4-2. `static/NBA.html` (필요 최소 범위)
- 구조 변경 없이 클래스 래퍼를 보강해 레이아웃 정렬 개선(필요 시만).
- 접근성 속성(`aria-selected`, `aria-hidden`) 동작은 기존 로직 유지.

### 4-3. `static/css/components/*.css` (필요 시)
- 공통 버튼/카드 규격과 충돌 시, 시장 전용 override보다 공통 토큰 확장을 우선 검토.
- 단, 다른 탭 UI 회귀를 막기 위해 범용 규칙 변경은 최소화.

## 5) 단계별 작업 순서
1. **비주얼 기준 추출**: 홈/메디컬에서 기준이 되는 색상·보더·radius·spacing 스펙 추출.
2. **시장 헤더/탭 정렬**: 헤더 카드화 + 서브탭 디자인 통일.
3. **FA/트레이드 블록 표 개선**: 테이블 헤더/행/빈 상태 스타일 개선.
4. **제안·협상 카드 개선**: 인박스 카드/액션 버튼/패키지 박스 리디자인.
5. **모달 개선**: 트레이드 딜 모달 시각 위계/상태 피드백 개선.
6. **반응형 점검**: 1366px, 1024px, 768px 기준으로 줄바꿈/오버플로우 검증.
7. **최종 QA**: 정보 무변경, 인터랙션 무회귀, 접근성 포커스 링 확인.

## 6) 품질 체크리스트 (완료 기준)
- [ ] 시장 탭 첫 인상이 홈/메디컬과 같은 제품군으로 인지된다.
- [ ] 기존 노출 정보(텍스트/수치/버튼 라벨/컬럼)가 1개도 삭제·추가되지 않는다.
- [ ] 모든 탭/서브탭/버튼에 hover/active/focus-visible 상태가 존재한다.
- [ ] 테이블 가독성(행 구분, 헤더 명확성, 정렬)이 개선된다.
- [ ] 빈 상태/로딩 상태/데이터 상태 모두 레이아웃 흔들림이 없다.
- [ ] 모달 열기/닫기/선택 인터랙션이 기존 기능과 동일하게 동작한다.

## 7) 리스크 및 대응
- **리스크**: market 전용 CSS 수정이 공통 버튼 스타일과 충돌할 수 있음.
  - **대응**: `#market-screen` 스코프를 우선 사용하고, 공통 컴포넌트 변경은 최소화.
- **리스크**: 라이트 테마 전환 시 대비(contrast) 저하 가능.
  - **대응**: 본문/보조/비활성 텍스트 대비를 WCAG 기준에 맞춰 재점검.
- **리스크**: 테이블 폭이 좁은 해상도에서 가로 스크롤 증가.
  - **대응**: 최소 폭 규칙 + 셀 패딩 미세 조정 + 우선순위 열 가독성 확보.
