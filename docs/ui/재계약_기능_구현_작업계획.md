# 재계약 기능 구현 작업계획 (선수 상세 > 계약 트랙)

## 0) 목표 / 범위

### 목표
- `선수 상세 정보` 화면의 `계약 트랙` 섹션에 `재계약 제의` 버튼을 추가한다.
- 버튼은 기본 비활성화이며, **계약 만료까지 1시즌 이하**일 때만 활성화한다.
- 버튼 클릭 시 `재계약 협상 화면`(1차 구현은 Player Detail 내부 협상 카드로 구현 가능)에 진입한다.
- 협상 화면에서 선수의 `ask_aav`, `ideal_years`를 보여주고,
  - 그대로 수락(빠른 제안) 또는
  - 연봉/기간 조정 제안(수동 제안)
  이 가능해야 한다.
- 유저 제안이 선수 수용 범위면 협상 성사(ACCEPTED) 및 재계약 커밋이 가능해야 한다.

### 이번 문서의 구현 범위
- UI/프론트 진입 및 상호작용
- 협상 API 연동 (`RE_SIGN` 모드)
- 조건부 활성화 규칙
- 성공/실패 UX

### 비범위 (후속)
- 협상 히스토리 상세 타임라인 고도화
- 고급 옵션(팀옵션/플레이어옵션) 편집 UI
- 다국어/i18n 정리

---

## 1) 핵심 규칙 정의 (판정 기준 고정)

## 1-1. 버튼 활성화 규칙
- 기본: `disabled = true`
- 활성 조건(모두 만족):
  1. 현재 팀 소속 선수(FA 아님)
  2. 활성 계약(`contract.active`) 존재
  3. `계약 종료 시즌 시작연도(end_start_year)`가 `현재 시즌 시작연도(current_start_year) + 1` 이하

> 예시: 현재 시즌이 `25-26`(start year = 2025)라면
> - 종료가 `25-26`(2025) 또는 `26-27`(2026)까지인 경우 활성
> - 종료가 `27-28`(2027) 이상이면 비활성

## 1-2. 종료 시즌 계산 기준
- 1순위: `contract.active.salary_by_year`의 최대 key 사용
- 보조: salary map이 비정상일 때 `start_season_year + years - 1`
- 둘 다 불가하면 비활성 유지 + 툴팁으로 이유 노출

## 1-3. 협상 성공 기준
- 백엔드 판단을 단일 진실원(SSOT)으로 사용
- 프론트는 ACCEPT/COUNTER/REJECT/WALK 결과만 표현
- `ACCEPTED` 상태에서만 커밋 버튼 활성

## 3) API 페이로드/상태전이 명세

## 3-1. 시작
- `POST /api/contracts/negotiation/start`
```json
{
  "team_id": "LAL",
  "player_id": "p_123",
  "mode": "RE_SIGN"
}
```

## 3-2. 오퍼 제출
- `POST /api/contracts/negotiation/offer`
```json
{
  "session_id": "...",
  "offer": {
    "start_season_year": 2026,
    "years": 2,
    "salary_by_year": {
      "2026": 28000000,
      "2027": 28000000
    },
    "options": []
  }
}
```

## 3-3. 카운터 수락
- `POST /api/contracts/negotiation/accept-counter`

## 3-4. 커밋
- 권장: `POST /api/contracts/re-sign-or-extend`
```json
{
  "session_id": "...",
  "team_id": "LAL",
  "player_id": "p_123"
}
```

## 5) 예외/에러 처리 정책

  - `NEGOTIATION_INVALID_MODE` → "재계약 대상 조건을 확인해주세요."

## 6) QA 시나리오 (필수)

1. **버튼 비활성 기본값**
   - 장기 계약(2시즌 초과 남음): 비활성
2. **버튼 활성 케이스**
   - 만료 시즌/다음 시즌 만료: 활성
3. **협상 시작**
   - 클릭 시 `RE_SIGN` 세션 생성, ask/ideal 노출
4. **그대로 제안**
   - ask/ideal 제출 시 ACCEPT 또는 COUNTER 정상 처리
5. **조정 제안**
   - 낮은 금액/기간 부적합 시 COUNTER/REJECT 확인
6. **커밋**
   - ACCEPTED에서만 재계약 확정 가능
7. **선수 전환 회귀**
   - 다른 선수 상세 진입 시 상태 혼선 없음
8. **기존 기능 회귀**
   - Market의 FA 협상 버튼/플로우 정상

## 8) 구현 메모 (개발자 주의)

- 시즌 계산은 숫자 비교 기준으로 고정 (`YYYY` start-year)하고, 라벨(`25-26`) 파싱 로직 중복을 피한다.
- 협상 수락 판단은 프론트 추정 금지. 백엔드 decision을 그대로 반영한다.
- 가능하면 협상 로직은 `myteam` 화면 코드와 분리해 `features/contracts`에 모듈화한다.
- 재계약 커밋 후에는 캐시/상세 화면을 반드시 갱신해 계약 트랙이 즉시 반영되게 한다.
