# 트레이드 블록 등록 모달 UI/UX 전면 개선 실행 계획 (2026-03-23)

## 1) 목표 요약
- 트레이드 블록 `등록` 모달의 선수 목록을 현재의 뒤섞인 버튼 나열 방식에서 **일렬(세로) 리스트**로 변경한다.
- 한 번에 여러 선수를 선택해 등록할 수 있도록 **다중 선택 UX + 다중 등록 API 호출 흐름**을 설계한다.
- 등록 모달 내 선수 정보에서 **OVR 노출을 제거**한다.
- 전체 디자인을 `시장` 탭의 밝고 정돈된 톤(카드형, 그라데이션, 라운드, 소프트 그림자)으로 통일해 **고퀄리티 라이트 모달**로 리디자인한다.

---

## 2) 수정 대상 파일과 역할

### A. `static/NBA.html`
**수정 목적:** 등록 모달 마크업 구조를 개선해 일렬 리스트/다중 선택/개선된 헤더&푸터 구성을 담는다.

예상 수정 포인트:
1. `#market-trade-block-roster-modal` 내부 구조를 확장
   - 현재 단순 `college-scout-options-list` 컨테이너를 다음 구조로 분리
     - 모달 헤더(타이틀/설명/선택 카운트)
     - 리스트 컨테이너(스크롤)
     - 액션 영역(닫기/선택 등록)
2. 접근성(ARIA) 보강
   - 리스트 항목: `aria-pressed` 또는 `aria-selected` 기반 토글 가능한 선택 상태 제공
   - 선택 개수 텍스트: `aria-live="polite"`로 갱신 가능하게 구성
3. 버튼 라벨 고도화
   - `등록` → `선택 선수 등록`처럼 다중 선택 의도를 명확히 반영

---

### B. `static/js/app/state.js`
**수정 목적:** 단일 선택 상태를 다중 선택 상태로 전환.

예상 수정 포인트:
1. 상태 필드 전환
   - `marketTradeBlockSelectedRosterPlayerId: null`
   - → `marketTradeBlockSelectedRosterPlayerIds: []` (또는 `Set` 직렬화 전략)
2. 모달 오픈/닫힘 시 선택 상태 초기화 규칙 명시
   - 오픈: 빈 배열로 초기화
   - 등록 성공 후: 빈 배열로 리셋
3. 선택 제한 정책(선택 가능 인원 상한)이 있다면 상수화
   - 예: `marketTradeBlockSelectionLimit` 도입(필요 시)

---

### C. `static/js/features/market/marketScreen.js`
**수정 목적:** 렌더링/이벤트/등록 로직을 다중 선택 기반으로 개편하고 OVR 노출 제거.

예상 수정 포인트:
1. `renderTradeBlockRosterModalList(rows)`
   - 기존 단일 선택 비교 로직(`selected === pid`) 제거
   - 다중 선택 배열/집합 기준으로 `is-selected` 판별
   - 항목 텍스트에서 `OVR` 제거
   - 항목 정보는 `선수명 + 포지션` 중심으로 단순화
   - 카드형/행형 마크업으로 전환(한 줄 1명)

2. 모달 리스트 클릭 이벤트(`els.marketTradeBlockRosterList?.addEventListener("click", ...)`)
   - 클릭 시 단일 대입이 아니라 `toggle` 처리
   - 이미 등록된 선수(`disabled`)는 토글 차단 유지
   - 토글 직후:
     - 리스트 재렌더
     - 선택 개수 UI 갱신
     - 확인 버튼 활성/비활성 갱신

3. 등록 API 호출 로직 분리
   - 기존 `listPlayerToTradeBlock(playerId)` 단건 호출 유지 + 배치 호출 래퍼 추가
   - 예: `listPlayersToTradeBlock(playerIds)`를 만들어 `Promise.allSettled`로 처리
   - 일부 성공/실패를 구분한 사용자 메시지 설계
     - 예: `3명 성공, 1명 실패` 형태

4. Confirm 버튼 핸들러 변경
   - 단일 `playerId` 검증 → 선택 배열 길이 검증
   - 다중 등록 완료 후 `loadTradeBlockMineList()` 1회 갱신
   - 완료 시 모달 닫기/알림 정책 정리

5. 모달 진입 함수(`openTradeBlockRosterModal`) 정비
   - 다중 선택 상태 초기화
   - 진입 시 선택 카운트/버튼 상태 초기 렌더

6. 보조 함수 도입(권장)
   - `updateTradeBlockRosterSelectionUi()`
   - `toggleTradeBlockRosterSelection(pid)`
   - `getTradeBlockRosterSelectedIds()`

### D. `static/css/screens/market.css`
**수정 목적:** 시장 탭의 라이트 톤 디자인 언어를 등록 모달에 이식.

예상 수정 포인트:
1. 등록 모달 전용 스코프 스타일 추가
   - 예: `#market-trade-block-roster-modal .confirm-modal-panel { ... }`
   - 기존 다크 인상(남색 배경 느낌)을 제거하고 밝은 카드형 톤으로 전환

2. 모달 패널/헤더/본문/푸터 디자인 고도화
   - 패널: 밝은 그라데이션 배경, 라운드 확대, 섀도우 정리
   - 헤더: eyebrow/타이틀/설명 간 간격 체계화
   - 본문: 최대 높이 + 내부 스크롤 영역(긴 로스터 대응)
   - 액션: 오른쪽 정렬 + 주요 액션 버튼 강조

3. 선수 리스트를 “일렬 리스트”로 시각 고정
   - `display: grid; grid-template-columns: 1fr;` 또는 `flex-direction: column`
   - 각 row 카드에 hover/selected/disabled 상태 분리
   - 선택 상태는 체크 인디케이터(아이콘/점/보더)로 직관화

4. 시장 탭 톤과 색상 일치
   - `#f8fbff`, `#ebf4ff`, `rgba(59, 130, 246, ...)` 계열을 우선 사용
   - 버튼/보더/포커스 스타일을 시장 탭 기존 패턴과 맞춤

5. 반응형 보완
   - 좁은 폭에서 버튼 영역 줄바꿈
   - 리스트 row의 텍스트 오버플로우 처리

## 4) 상세 기능 시나리오 (완료 기준)
1. `등록` 버튼 클릭 시 밝은 톤의 모달 오픈.
2. 선수 목록은 한 줄에 한 명씩 정렬되어 스크롤 가능.
3. 여러 선수를 클릭해 동시에 선택 가능.
4. 각 선수는 `이름 + 포지션`만 노출하며 OVR은 표시하지 않음.
5. 하단 CTA 클릭 시 선택된 선수들이 일괄 등록.
6. 처리 결과(성공/실패 수)가 사용자에게 명확히 표시.
7. 등록 완료 후 내 팀 트레이드 블록 리스트가 새로고침됨.

## 5) 리스크 및 대응
- **리스크:** 공통 `.confirm-modal` 스타일과 충돌해 다른 모달 디자인이 깨질 수 있음.
  - **대응:** `#market-trade-block-roster-modal` 하위 스코프 스타일로 제한.

- **리스크:** 다건 등록 시 네트워크 실패가 일부 선수에만 발생.
  - **대응:** `Promise.allSettled`로 부분 실패를 수집하고 요약 메시지 제공.

- **리스크:** 리스트 재렌더 빈도가 높아 UX가 끊길 수 있음.
  - **대응:** 토글 시 최소 DOM 업데이트(필요 시 이벤트 위임 + 부분 업데이트).

## 6) 테스트/검증 체크리스트
- [ ] 등록 모달 진입 시 선택 상태가 항상 초기화된다.
- [ ] 0명 선택 상태에서 등록 버튼이 비활성 또는 경고 처리된다.
- [ ] 2명 이상 선택 후 등록 시 모두 서버 요청이 전송된다.
- [ ] 이미 등록된 선수는 disabled 상태이며 선택되지 않는다.
- [ ] 리스트 어디에도 `OVR` 문자열이 노출되지 않는다.
- [ ] 모달 디자인이 시장 탭과 동일한 라이트 톤/고품질 카드 스타일을 유지한다.
- [ ] 모바일/협소 해상도에서 레이아웃이 깨지지 않는다.
