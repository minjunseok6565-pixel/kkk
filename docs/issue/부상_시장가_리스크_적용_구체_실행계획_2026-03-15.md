작성일: 2026-03-15  
대상 범위: `trades/valuation` (market_pricing + team_utility + service/config + tests)

## 0) 목표 요약

이 문서는 트레이드 valuation의 아래 2개 이슈를 동시 개선하기 위한 **구현 가능한 최종 형태**와 **파일별 작업 목록**을 정의한다.

- 이슈 7) 선수 시장가가 OVR/나이 중심으로 과도 단순화
- 이슈 10) 리스크 할인이 나이+계약연차 2변수 합성에 머묾

핵심 방향:
1. 부상 신호를 **현재 상태(current)** 와 **이력(history)** 로 분리.
2. 하드 임계치 대신 **소프트 게이트(sigmoid/포화함수)** 를 사용.
3. 가치 하락만이 아니라 **건강 보너스(health credit)** 를 함께 도입.
4. 모든 수치(경계/민감도/상한)는 config/trade_rules로 노출해 튜닝 가능하게 설계.

## 1) 최종 목표 형태(정의)

## 1-1) 시장가(Market Price) 최종 형태

선수 시장가의 basketball component에 아래를 순차 적용한다.

\[
V_{market}^{player}
= (V_{base} \times F_{inj,current} \times F_{inj,history}) + B_{health}
\]

- `V_base`: 기존 OVR/age/position/contract 기반 값(현행 유지)
- `F_inj,current`: 현재 부상 할인 팩터 (0~1)
- `F_inj,history`: 부상 이력 할인 팩터 (0~1)
- `B_health`: 건강 보너스 (>=0, 제한된 상한)

### A. 현재 부상 할인 (요구사항 반영)
요구사항:
- 복귀까지 1개월 이내: 미미한 감소
- 1~6개월: 완만/선형에 가까운 감소
- 6개월 초과: 큰 감소
- 경계값 하드 분기 금지

권장 함수:
- `d = days_to_return`
- `g30 = sigmoid((d - t30) / s30)`
- `g180 = sigmoid((d - t180) / s180)`
- `mid = clamp((d - t30) / max(t180 - t30, eps), 0, 1)`
- `curr_penalty = w30 * g30 * mid + w180 * g180`
- `F_inj,current = clamp(1 - curr_penalty, current_factor_floor, 1)`

초기 권장값:
- `t30=30`, `t180=180`
- `s30=8~14`, `s180=18~30`
- `w30=0.05~0.10`, `w180=0.08~0.20`

### B. 이력 할인 (요구사항 반영)
요구사항:
- 최근 부상 잦을수록 감소
- 치명 부위 부상 잦을수록 감소
- 동일 부위 반복이 잦을수록 감소
- 하드코딩 감점표 금지

피처:
- `recent_180 = history.recent_count_180d`
- `critical_365 = history.critical_count_365d`
- `repeat_same = max(history.same_part_repeat_365d_max - 1, 0)`
- `sev_w = history.weighted_severity_365d`

소프트 합성:
- `soft_count(x,k)=1-exp(-k*x)`
- `fr = soft_count(recent_180, kr)`
- `fc = soft_count(critical_365, kc)`
- `fp = soft_count(repeat_same, kp)`
- `fs = soft_count(max(sev_w-1,0), ks)`
- `hist_penalty = cap_hist * (ar*fr + ac*fc + ap*fp + as*fs)`
- `F_inj,history = clamp(1 - hist_penalty, history_factor_floor, 1)`

초기 권장:
- critical/repeat 가중치를 recent보다 높게 (`ac, ap > ar`)
- `cap_hist`로 상한 제한(예: 0.18~0.25)

### C. 건강 보너스(반대급부)
요구사항:
- 건강할수록 가치 상승을 부여
- 다만 악용 방지를 위해 할인보다 약하게

권장:
- `avail = health_credit_inputs.availability_rate_365d`
- `critical_365 == 0`일 때 보너스 가중 증가
- `bonus_raw = softplus((avail - avail_ref)/avail_scale)` 또는 `sigmoid` 기반
- `B_health = min(health_credit_cap, bonus_raw * base_scale)`

권장 상한:
- `health_credit_cap <= 할인상한의 1/3~1/2`

## 1-2) 리스크 할인(Team Utility) 최종 형태

현재 age/term 2축을 확장하여 injury 축을 포함한다.

\[
R = squash( w_{at}R_{age\_term} + w_hR_{inj\_hist} + w_cR_{inj\_current} )
\]
\[
F_{risk}= clamp(1 - scale \cdot risk\_mult \cdot R, floor, cap)
\]

- `R_age_term`: 현행 age/contract years 기반 리스크
- `R_inj_hist`: 이력 기반 리스크
- `R_inj_current`: 현재 OUT/RETURNING + 부위/중증도 기반 리스크
- `squash`: sigmoid/tanh로 0..1 고정

### A. 이력 리스크
- `recent_180`, `critical_365`, `repeat_same`, `weighted_severity_365d`를 soft-count로 변환
- critical/repeat에 높은 가중

### B. 현재 리스크
- 현재 `status`가 OUT/RETURNING이면 활성
- `current.body_part`가 critical set이면 추가 가중
- `current.severity`가 높을수록 가중

예시 합성:
- `r_cur = gate_status * (b0 + b_part*is_critical_part + b_sev*sev_norm + b_days*g30_or_g180)`
- `R_inj_current = clamp(r_cur, 0, 1)`

### C. 건강 선수 완화
- availability 높고 최근 critical/repeat가 낮으면 `R` 완화 항 추가
- 단, 완화는 할인 반전이 아닌 **리스크 축소** 수준으로 제한

## 3) 튜닝 파라미터 초안

초기값은 보수적으로 시작하고 로그 기반 캘리브레이션한다.

- 시장가
  - current injury max penalty: 8~18%
  - history max penalty: 10~22%
  - health credit max bonus: 3~7%
- 리스크 할인
  - injury hist/current 합산 비중: 전체 risk의 35~55%
  - 단일 신호 cap 및 squash로 과도한 급락 방지

## 6) 완료 판정 기준 (DoD)

1. 시장가가 부상 current/history/health credit 3요소를 모두 반영한다.
2. 리스크 할인이 age/term + injury hist/current + health relief 다축을 반영한다.
3. 경계(1개월/6개월)에서 결과가 연속적이며 급변하지 않는다.
4. critical/repeat가 증가할수록 점진적 감점(단조성)이 보장된다.
5. 건강 보너스는 존재하되 할인 총량을 압도하지 않는다.
6. 모든 신규 단계가 로그(meta)로 설명 가능하다.
7. 기존 세이브에서 파라미터 미지정 시 backward-compatible default로 동작한다.
