# 경기 진행/자동 진행 플로우 검토 (2026-03-05)

## 목표
프론트엔드에서 아래 두 버튼 플로우를 안전하게 구현한다.

1. `경기 진행`
   - 당일에 유저 팀 경기가 있으면: 유저 팀 경기 + 같은 날짜의 타팀 경기까지 모두 진행.
   - 당일에 유저 팀 경기가 없으면: 경고 확인 후, 다음 유저 경기 전날까지 타팀 경기 자동 진행 + 유저 경기 당일(유저 경기 + 타팀 경기) 진행.

2. `경기 날짜까지 자동 진행`
   - 현재 날짜부터 유저 팀 다음 경기 전날까지 타팀 경기 자동 진행.
   - 날짜를 유저 경기일로 맞춘 뒤 대기.
   - 이후 `경기 진행` 클릭 시 유저 경기일 전체(유저+타팀) 진행.

---

## 현재 백엔드 API/엔진 동작 핵심

### 1) `/api/simulate-game`
- 지정한 **한 경기**만 시뮬레이션한다.
- 스케줄 기반 매칭(날짜+home+away)이 맞아야 하며, 이미 final인 게임은 거부한다.
- 이 API만으로는 같은 날짜의 다른 경기들이 자동으로 같이 돌지 않는다.

### 2) `/api/advance-league`
- `current_date + 1`부터 `target_date`까지 경기 순회.
- `user_team_id`를 넘기면 유저 팀이 포함된 경기는 건너뛰고 타팀 경기만 자동 진행.
- 루프 종료 시 `current_date = target_date`로 설정.

### 3) 중요한 제약
- `advance-league`는 루프 시작이 항상 `current_date + 1`이므로,
  - `current_date`가 이미 유저 경기 당일이면 `target_date=당일` 호출 시 **아무 경기도 돌지 않음**.
- 따라서 "유저 경기 당일의 타팀 경기"까지 묶어 처리하려면,
  - 별도 백엔드 오케스트레이션 엔드포인트가 필요하거나,
  - 프론트에서 날짜 이동/호출 순서를 매우 엄격히 제어해야 한다.

---

## 결론: 현재 API만으로는 요구사항을 100% 안정적으로 만족시키기 어렵다

특히 아래 케이스에서 누락 위험이 있다.

- "오늘 유저 경기 있음" 상태에서 `simulate-game`을 먼저 호출하면,
  - 같은 날짜 타팀 경기를 뒤늦게 `advance-league(target=today)`로 처리할 수 없음
  - (루프 시작이 current+1 규칙).

즉, **당일 전체 슬레이트(유저+타팀) 원자 처리**가 보장되지 않는다.

## 권장 구현안 (누락 방지)

## A안 (권장): 신규 오케스트레이션 API 2개 추가

### A-1) `POST /api/game/progress-next-user-game-day`
한 번의 호출로 아래를 원자적으로 수행:

1. 현재 날짜 확인
2. 유저 팀 다음 경기일 `D` 탐색
3. `current_date+1 ~ D-1`: 타팀 경기 자동 진행 (`skip user team`)
4. 날짜 `D`에서
   - 유저 팀 경기 1건 진행
   - 같은 날짜 타팀 경기 모두 진행
5. `current_date = D` 또는 정책상 `D+0` 유지

> 프론트의 `경기 진행` 버튼은 이 API 하나만 호출하면 됨.

### A-2) `POST /api/game/auto-advance-to-next-user-game-day`
한 번의 호출로 아래를 수행:

1. 유저 팀 다음 경기일 `D` 탐색
2. `current_date+1 ~ D-1` 타팀 경기 자동 진행
3. 날짜를 `D`로 세팅 (경기는 미진행)
4. 요약 반환 (`advanced_games_count`, `next_game_date=D`)

> 프론트의 `경기 날짜까지 자동 진행` 버튼은 이 API 호출 + 경고 모달 UX로 연결.

---

## B안 (차선): 기존 API 조합

가능하긴 하나, 다음 이유로 실수 가능성이 높다.

- 날짜 기준 분기/다중 호출 순서가 복잡
- 현재 날짜가 당일인지 전날인지에 따라 결과가 달라짐
- 동시성(중복 클릭) 보호를 프론트에서 강하게 해야 함

운영 안정성 관점에서 A안이 훨씬 안전하다.

## 누락 방지 검증 체크리스트

아래 테스트를 통과하면 "누락 없이 진행" 신뢰도를 확보할 수 있다.

1. **당일 유저 경기 케이스**
   - 버튼 1회 클릭 후
   - 해당 날짜 모든 게임 status=final
   - 유저 경기 결과 1건만 생성(중복 없음)

2. **유저 경기 없는 날 케이스(다음 경기 3일 후 등)**
   - 경고 확인 후 실행
   - 중간 날짜의 타팀 경기 모두 final
   - 유저 경기일 전체 슬레이트 final

3. **자동 진행 버튼 후 경기 진행 버튼 케이스**
   - 자동 진행 완료 직후 current_date=유저 경기일
   - 이후 경기 진행 클릭 시 유저 경기 + 동날짜 타팀 경기 final

4. **더블 클릭/중복 요청 방지**
   - 동일 요청 연속 호출 시도 시 결과 일관성 보장
   - 이미 final 게임 재시뮬레이션 차단

5. **시즌 경계/오프시즌/포스트시즌 전환일**
   - 정규시즌 종료 인접 날짜에서도 예외 없이 종료

## 프론트 UX 연결 제안

- `경기 진행`
  - 서버가 반환한 `mode`가 `auto_advanced_then_played`면
    - "경기 날짜까지 자동 진행 후 당일 경기를 완료했습니다" 토스트
- `경기 날짜까지 자동 진행`
  - 경고 문구: "경기 날짜까지 리그 일정은 자동 진행됩니다"
  - 확인 시 A-2 API 호출
  - 완료 토스트: "다음 경기일로 이동했습니다. 이제 경기 진행을 눌러 경기를 시작하세요"

## 최종 판단

- 요청하신 제품 플로우는 **충분히 실현 가능**하다.
- 다만 현재 API 조합만으로는 당일 타팀 경기 누락 가능성이 있어,
  **백엔드 오케스트레이션 API 추가(A안)**를 권장한다.
- A안으로 구현하면 유저 팀/타팀 경기 누락 없이 일관된 진행을 보장하기 쉽다.
