# NBA 시뮬레이션 게임 메인 화면 고도화 기획안 (상업용 출시 품질)

## 1) 문서 목적 / 범위
- 목적: `static/NBA.html`, `static/NBA.css`, `static/NBA.js`만으로 메인 화면(Home 탭)을 상업용 게임 품질로 재설계하기 위한 실행 가능한 기획 기준 제공.
- 범위: **메인 화면만 개선**. 내 팀/전술/스케줄/훈련/순위/대학/시장/메디컬 센터 등 다른 화면 및 라우팅 구조는 수정 금지.
- 데이터 제약: 새로운 API 추가 금지. 기존 조회 API의 응답값만 조합/가공/재배치하여 정보 밀도와 인사이트를 강화.

## 2) 품질 목표 (Definition of Premium)

### 2-1. 비주얼 목표
1. 첫 진입 3초 내에 “팀 현재 상태 + 다음 액션”이 파악될 것.
2. 프로토타입/바이브 코딩 느낌을 제거할 것(컴포넌트 일관성, 간격 체계, 인터랙션 디테일).
3. 스포츠 게임 특유의 긴장감(매치업, 일정 압박, 컨디션 리스크)이 시각적으로 전달될 것.

### 2-2. UX 목표
1. 홈 화면은 “상태 확인”이 아니라 “결정 지원” 중심으로 설계.
2. 중요한 알림(부상/피로/백투백/연패)을 최상단 우선순위로 노출.
3. 주요 CTA는 명확히 1순위(`경기 진행`) + 보조 CTA(`전술`, `빠른 진행`)로 역할 분리.

### 2-3. 엔지니어링 목표
1. HTML 구조를 semantic section 단위로 분리해 유지보수성 향상.
2. CSS를 토큰화(색/간격/타이포/radius/shadow)하여 임의 스타일 누적 방지.
3. JS는 데이터 바인딩과 UI 상태(로딩/에러/빈상태/정상)를 명확히 분리.

## 3) 메인 화면 정보 구조(IA) 재설계

## 3-1. 화면 전체 구조 (상단→하단)
1. Global Top Nav (기존 유지, 스타일 고급화)
2. Franchise Command Bar (신규)

4. Action Intelligence Row (신규 2열)
5. Weekly Timeline + Activity Feed Row (신규 2열)
6. Sticky Action Footer (기존 하단 버튼 그룹 고도화)

## 4) 섹션별 상세 기획

## 4-1. Global Top Nav (기존 기능 유지)
- 기능: 탭 이동 구조는 그대로 유지(정보구조 변경 금지).
- 디자인:
  - 배경: 반투명 다크 레이어 + 얇은 하단 보더
  - 현재 탭 강조: 밑줄 + 컬러 바 + medium weight
  - hover/focus 상태 추가(키보드 접근성 포함)
- 기대효과: 첫 줄부터 완성도와 통일감을 형성.

## 4-2. Franchise Command Bar (신규)
- 배치: 네비게이션 바로 아래, 가로 1줄 카드.
- 좌측 정보:
  - 팀명(예: 골든 스테이트 워리어스)
  - 시즌 단계/현재 날짜(기존 조회값)
- 중앙 KPI(4칸):
  1) 시즌 성적(승-패)
  2) 승률
  3) 최근 10경기(예: 7-3)
  4) 주요 위험 배지(부상자 수, 피로 주의 인원)
- 우측 액션:
  - 빠른 저장, 설정(아이콘 + 텍스트)
- 데이터 규칙:
  - 응답값 누락 시 `--` 처리
  - 위험 지표는 임계치 기반 배지 표기 (`정상/주의/위험`)

## 4-3. Hero: Next Game Cinematic Card (핵심)
- 목표: 현재 화면의 VS 정보를 “결정 중심 프리뷰”로 승격.
- 구성:
  1) 상단 타이틀: `NEXT GAME`
  2) 중앙: 홈/원정 팀 로고 + 팀명 + VS
  3) 경기 메타: 경기시간, 경기장(있으면), 홈/원정 상태
  4) 하단 탭 3개(클라이언트 탭 전환)
     - 탭 A: 매치업 비교
       - 팀 종합 지표(예: 공격/수비/리바운드 등 기존 조회 데이터 기반)
       - 상대 대비 ± 표시(막대/칩)
     - 탭 B: 컨디션 리포트
       - 출전 가능/주의/결장 리스트
       - 피로도 높은 선수 Top N
     - 탭 C: 전략 힌트
       - 데이터 기반 문장 2~3줄 (정적 템플릿 + 수치 삽입)
- 인터랙션:
  - 탭 전환 150ms fade/slide
  - 버튼 hover 시 elevation + outline
- 주의:
  - 탭 콘텐츠는 모두 기존 조회 응답 조합으로 생성.

## 4-4. Action Intelligence Row (신규)

### 좌측: 오늘의 우선 과제 (Primary Decision Panel)
- 예시 블록:
  - 일정 리스크: 백투백 여부, 최근 N일 경기 수
  - 로스터 리스크: 주전 피로/부상
  - 경기 준비: 상대 상성 기반 권장 전술
- CTA:
  - `전술로 이동`
  - `훈련 조정`
  - `메디컬 확인`
- 표시 규칙:
  - 중요도 순으로 최대 3건만 노출
  - 긴 문장 금지(한 줄 요약 + 세부값)

### 우측: Quick Intel Stack
- 카드 3개 스택:
  1) 부상/복귀 일정
  2) 유망주/벤치 성장 알림
  3) 시장/트레이드 관심 신호
- 모두 기존 조회 데이터에서 파생해 표시(없으면 empty state).

## 4-5. Weekly Timeline + Activity Feed Row (신규)

### 좌측: 주간 타임라인
- 날짜별 이벤트 칩(경기/휴식/이동/백투백)
- 다음 경기일은 강조 표시
- 연속 원정/강행군 구간 색상 강조

### 우측: 프랜차이즈 활동 피드
- 최근 이벤트 역순:
  - 최근 경기 결과
  - 부상 발생/복귀
  - 로테이션 변경
- 각 아이템에 유형 아이콘 + 타임스탬프 + 상태 배지

## 4-6. Sticky Action Footer (기존 버튼 영역 고도화)
- 버튼 계층:
  - Primary: `경기 진행`
  - Secondary: `전술`
  - Ghost: `빠른 진행`
- 규칙:
  - 모바일/저해상도에서 2열 접힘 대응
  - 비활성 조건(데이터 미로딩/필수 조건 미충족) 명확히 표기

## 5) 디자인 시스템 (메인 화면 한정 토큰)

## 5-1. Color Tokens
- `--bg-base`: #0E1117 (다크 베이스)
- `--bg-elev-1`: #151A23
- `--bg-elev-2`: #1B2130
- `--text-primary`: #F5F7FA
- `--text-secondary`: #A8B1C3
- `--line-subtle`: rgba(255,255,255,0.08)
- `--accent-team`: 팀 컬러 동적 적용
- 상태색:
  - `--state-good`: #35C759
  - `--state-warn`: #FFB020
  - `--state-danger`: #FF5A5F

## 5-2. Typography Scale
- Display: 40/48, 700
- H1: 30/38, 700
- H2: 24/32, 700
- H3: 20/28, 600
- Body: 15/24, 400~500
- Meta: 12/18, 500
- 숫자/기록: tabular 느낌 정렬(가능 시 폰트 feature)

## 5-3. Spacing & Radius
- spacing: 4, 8, 12, 16, 24, 32, 40, 48
- radius:
  - 카드 16
  - 입력/버튼 12
  - 배지 9999(pill)

## 5-4. Shadow & Border
- 카드 기본: 1px subtle border + low blur shadow
- hover: shadow intensity +10~15%

## 6) 컴포넌트 사양

## 6-1. Card 공통
- 구조: Header / Body / Footer(optional)
- 내부 패딩: 20~24px
- Header에는 항상 제목 + 우측 보조정보(있으면)
- 빈 상태: 아이콘 + 1줄 메시지 + 보조 설명

## 6-2. Status Badge
- 종류: `GOOD`, `WARN`, `RISK`, `OUT`
- 크기: height 22, 폰트 11~12
- 색상은 상태 토큰만 사용(임의 hex 금지)

## 6-3. CTA Button
- 높이 48 고정
- 좌우 패딩 20
- disabled, hover, focus-visible 상태 필수
- primary는 단일 강포인트 컬러만 사용

## 6-4. Mini Stat Item
- Label + Value + Delta(선택)
- Delta는 방향 아이콘(▲▼) 또는 색상으로만 표현

## 7) 데이터 매핑 가이드 (신규 API 금지 준수)

> 아래는 “기존 조회 API 응답을 어떻게 재가공할지”에 대한 화면용 규칙이다.

1. `다음 경기 카드`
   - 기존 일정 조회의 다음 경기 1건 사용
   - 홈/원정, 상대팀명, 경기시간 노출
2. `KPI`
   - standings/team summary 값 활용
   - 최근 N경기 record는 기존 결과 로그 집계
3. `컨디션 리포트`
   - 메디컬/피로 조회값 결합
4. `전략 힌트`
   - 전술/상대 지표 비교값으로 템플릿 문장 생성
   - 예: “상대 페인트존 득점 비중이 높음 → 인사이드 수비 강화 권장”
5. `활동 피드`
   - 최근 이벤트 목록을 유형별 normalize 후 시간 역순 표시

## 8) 상태별 UX 정의

## 8-1. Loading
- 섹션별 skeleton 적용(카드 형태 유지)
- 글로벌 스피너 남발 금지

## 8-2. Empty
- 예: 부상자 없음, 이벤트 없음
- 긍정 메시지 + 정보 없는 이유 명시

## 8-3. Error
- 카드 단위 에러 문구 + 재시도 액션
- 전체 화면 블랭크 금지

## 8-4. Partial Failure
- 일부 API 실패 시 성공 섹션 우선 렌더
- 실패한 카드만 degraded state 표시

---

## 9) 모션/인터랙션 가이드
- duration: 120~180ms (기본 150ms)
- easing: `cubic-bezier(0.2, 0.8, 0.2, 1)`
- hover: transform Y -2px 이내
- 탭 전환: opacity + translateY 4px
- reduced motion 환경에서는 애니메이션 최소화

## 11) 수용 기준 (Acceptance Criteria)
1. 메인 화면 진입 시 상단 1스크린 내에서 현재 팀 상태 + 다음 경기 + 핵심 행동 CTA가 모두 보인다.
2. 중요한 리스크(부상/피로/백투백)가 1회 이상 시각적으로 강조된다.
3. 버튼/카드/배지 스타일이 화면 전반에서 일관된다.
4. 데이터 누락/로딩/오류 시 레이아웃 붕괴 없이 동작한다.
5. 다른 메뉴 화면 구조/동작에는 영향이 없다.

## 14) 결론
이 기획안의 핵심은 “화려한 장식”이 아니라 “게임 운영 결정을 돕는 정보 디자인”이다. 
기존 조회 API만으로도 레이아웃 재설계, 시각적 위계, 상태 중심 UX를 적용하면 상업용 출시 수준의 메인 화면 완성도가 가능하다.
