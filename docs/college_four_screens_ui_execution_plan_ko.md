# 대학 리그 4개 화면 UI 실행 기획안 (상업용 품질 목표)

## 0) 문서 목적
이 문서는 **대학 순위 / 대학 선수 리더보드 / 빅보드 / 스카우트** 4개 화면만 대상으로, 다음 개발 턴에서 바로 구현 가능한 수준의 프론트엔드 실행 기획안을 정의한다.

- 대상 파일(구현 범위): `static/NBA.html`, `static/NBA.css`, `static/NBA.js`
- 절대 비대상: 위 4개 화면 외 다른 화면/기능/도메인 로직
- 데이터 제약: **기존 조회 API/기존 응답 필드만 사용**, 신규 API 추가 금지

---

## 1) 제품 품질 목표 (Release Bar)

### 1-1. 상업용 UI 품질 기준
1. 첫 진입 3초 내에 “현재 맥락(시즌/리그 상태/탭 위치)”이 파악되어야 한다.
2. 핵심 작업(정렬, 선수 선택, 스카우트 배정)이 1~2클릭 내 완료되어야 한다.
3. 테이블 중심 화면이어도 “게임스러운 정보 연출(계층/상태/강조)”이 명확해야 한다.
4. 빈 상태/로딩/성공/실패 피드백이 모두 정의되어야 한다.

### 1-2. 성공 KPI (정성)
- “엑셀 느낌” 감소, “프런트오피스 콘솔 느낌” 강화
- 동일 정보량 대비 탐색 시간 단축
- 탭 간 일관성(타이포/간격/상호작용/색 체계) 확보

---

## 2) 작업 범위 정의 (In/Out)

## In Scope
- 4개 탭의 레이아웃, 타이포, 컬러, 컴포넌트 상태, 마이크로 인터랙션
- 기존 데이터 조합 기반 파생 표시(클라이언트 계산)
- 접근성 기본 개선(키보드 포커스, 대비, aria-label 보완)

## Out of Scope
- 백엔드 엔드포인트 추가/응답 스키마 변경
- 신규 게임 로직/시뮬레이션 로직
- 4개 탭 외 페이지의 디자인 변경

## 3) 공통 UX/비주얼 시스템

## 3-1. 레이아웃 시스템
- 콘텐츠 최대 너비: 1340~1440px
- 12컬럼 그리드 기준
  - 메인 테이블 영역: 8~9컬럼
  - 우측 상세/보조 영역: 3~4컬럼
- 간격 스케일 고정: `4 / 8 / 12 / 16 / 24 / 32`
- 카드 기본 규격
  - radius: 14px
  - padding: 20px
  - border: 1px solid 중저명도
  - shadow: 매우 약한 레이어 1단

## 3-2. 타이포 시스템
- 헤더(H1/H2): 28/22px, 700
- 섹션 타이틀: 18px, 650
- 테이블 헤더: 13px, 600, 대문자/트래킹 소폭
- 본문: 14px, 450~500
- 숫자 셀: tabular-nums 적용

## 3-3. 컬러 토큰
- `--bg-page`: 전체 배경
- `--bg-card`: 카드 배경
- `--text-primary`, `--text-secondary`, `--text-muted`
- `--line-default`, `--line-strong`
- `--accent-primary` (브랜드 포인트)
- 상태색: `--state-up`, `--state-down`, `--state-neutral`
- 티어색: `--tier-elite`, `--tier-lottery`, `--tier-round1`, `--tier-round2`

## 3-4. 상호작용 시스템
- hover 전환 120~160ms
- 탭 전환 180~220ms (opacity + translateY)
- 정렬 아이콘 회전/방향 전환 120ms
- 행 선택 시 좌측 인디케이터 바 + 우측 상세 패널 동기화
- 버튼 상태: default / hover / pressed / disabled / focus-visible 모두 정의

## 3-5. 상태 정의
- Loading: 스켈레톤 행 6~10개
- Empty: 안내 문구 + 다음 행동 CTA
- Error: 짧은 원인 + 재시도 버튼
- Success: 토스트(자동 사라짐 2.2s)

## 4) 데이터 활용 원칙 (신규 API 금지 대응)

## 4-1. 사용 가능 데이터 소스
- 현재 각 탭에서 이미 조회해 렌더링 중인 리스트/필드
- 선택된 팀/선수/스카우트 상태 정보

## 4-2. 클라이언트 파생 계산 허용 항목
- 평균/합계/비율 (예: 팀 로스터 평균 PTS)
- 분포 집계 (예: POS 분포, 학년 분포)
- 단순 지표(가중합 기반)
- 문자열 파싱(빅보드 요약의 Strength/Concern 태그화)

## 4-3. 금지 항목
- 서버 저장을 전제로 한 신규 필드
- API 미존재 데이터 가정 UI(가짜 값 하드코딩)

## 5) 화면별 상세 기획

## A. 대학 팀 순위 탭

### A-1. 목적
- 리그 판도(상위권/컨퍼런스 구조)와 선택 팀 로스터를 동시에 탐색

### A-2. 정보 구조
1. 상단 컨텍스트 바
   - 시즌, 대학팀 수, 예정 드래프트 연도
2. 리그 스냅샷 카드(3개)
   - 최고 SRS 팀
   - 컨퍼런스별 TOP팀 수
   - 전체 평균 SRS
3. 메인 팀 순위 테이블
4. 우측 팀 상세/로스터 패널

### A-3. 테이블 컬럼 연출
- RK: 상위권(1~4) 시드 배지
- 팀명: 굵기 강화 + 선택 시 하이라이트
- SRS: 수치 + 미니 바(상대 스케일)
- W/L: 시즌 초기 0이라도 자리 유지(추후 변화 대비)

### A-4. 우측 상세 패널
- 팀명 + 컨퍼런스 + W/L/SRS 헤더
- 포지션 분포 칩(G/W/B 또는 PG~C 집계)
- 학년 분포(1~4)
- 로스터 테이블(현재 구조 유지, 가독성 강화)

### A-5. 사용자 체감 포인트
- 팀 클릭 시 우측 패널 즉시 갱신 + 짧은 하이라이트
- 상위권 팀 시각 강조로 “리그 경쟁” 체감 강화

---

## B. 대학 선수 리더보드 탭

### B-1. 목적
- 단순 스탯 나열이 아닌 “왜 상위인지”를 즉시 이해

### B-2. 정보 구조
1. 상단 툴바
   - 정렬 기준(기존)
   - 포지션 필터(클라이언트)
   - 팀 필터(클라이언트)
   - 검색(선수명 contains)
2. 리더보드 테이블
3. 우측 선수 인사이트 패널

### B-3. 테이블 연출
- 상위 10위 행 미세 강조
- 선수 셀: 이름 + 팀명 보조 텍스트 + POS 배지
- 숫자 열 오른쪽 정렬 통일

### B-4. 인사이트 패널
- 선택 선수 핵심 스탯 3개 카드
- 현재 정렬 기준 순위 표시
- 팀 내 비교(동일 팀 선수 중 해당 스탯 순위)

### B-5. 파생 지표(선택)
- `Impact Index = 0.5*PTS + 0.25*REB + 0.25*AST`
- UI 라벨에 “커스텀 지표” 명시하여 혼동 방지

---

## C. 빅보드 탭

### C-1. 목적
- 표가 아닌 “드래프트 보드” 경험 제공

### C-2. 정보 구조
1. 상단 보드 인사이트
   - Tier 1 인원
   - Lottery 인원
   - POS 분포
2. 빅보드 테이블
3. 우측 선수 디테일 카드

### C-3. 테이블 연출
- Tier 텍스트 → 컬러 칩
- 요약 문장 파싱:
  - Strength 태그(긍정 컬러)
  - Concern 태그(주의 컬러)
- 랭크 상위권 미세 배경 강조

### C-4. 디테일 카드
- 강점 Top 3, 우려 Top 2 태그 강조
- 동일 포지션 내 현재 보드 순위
- “Scout Note” 스타일의 서술형 영역(기존 요약 재배치)

### C-5. 정렬 UX
- 전문가/정렬 옵션 변경 시 헤더 상태 배지 업데이트
- 정렬 기준 툴팁 제공(데이터형/효율형 의미)

---

## D. 스카우트 탭

### D-1. 목적
- 단순 배정 폼 → 운영 대시보드 체감

### D-2. 정보 구조
1. 좌측: 배정 컨트롤 카드
2. 우측: 리포트 피드 카드
3. 하단(여유 시): 배정 현황 요약 바

### D-3. 배정 컨트롤 카드
- 스카우터 선택, 대상 선수 선택(현행 유지)
- 선택 즉시 퀵 프리뷰:
  - 선수명/팀/포지션
  - 현재 보드 구간(가능 시)
- CTA
  - `배정` Primary
  - `해제` Secondary/Danger

### D-4. 리포트 피드 카드
- 컬럼 구조 유지하되 행 높이/정렬 개선
- 빈 상태 메시지 개선:
  - “리포트 없음” + “생성 조건/시점” 안내
- 상태 배지(완료/진행/대기) 시각 분리

### D-5. 피드백
- 배정/해제 성공 토스트
- 중복 배정 불가 등 제약 발생 시 인라인 경고

## 6) 컴포넌트 명세 (구현 체크리스트)

## 6-1. Tabs
- 높이 40~44px, radius 10px
- active 탭: 배경 + 그림자 + 텍스트 강조
- 키보드 좌우 이동 지원

## 6-2. Data Table
- header sticky(가능하면)
- row height 42~46px
- zebra는 미세하게, hover 대비 우선
- 정렬 가능한 헤더는 커서/아이콘/aria-sort 제공

## 6-3. Badge / Chip
- 용도 분리: Tier, POS, Status
- 최소 28px 높이, 좌우 10~12px 패딩

## 6-4. Toast
- 우상단 stack, 최대 3개
- 성공/경고/오류 컬러 구분

## 6-5. Empty State
- 타이틀 + 설명 + 행동 버튼(선택)
- 화면별 톤앤매너 통일

## 7) 구현 단계 (다음 턴 개발 순서)

### Phase 1 — 기반 정리 (반드시 선행)
1. CSS 토큰/타이포/간격 시스템 도입
2. 공통 카드/테이블/탭 스타일 리팩터
3. 상태(hover/focus/disabled) 정리

### Phase 2 — 화면 구조 개선
1. 팀 순위 탭: 스냅샷 카드 + 우측 상세 강화
2. 리더보드 탭: 필터 바 + 인사이트 패널
3. 빅보드 탭: Tier/태그 시각화
4. 스카우트 탭: 운영형 배정 카드 + 리포트 상태

### Phase 3 — 완성도/폴리싱
1. 모션/전환 미세 튜닝
2. 빈 상태/오류 상태 문구 품질 상향
3. 접근성 라벨/포커스 링/대비 최종 점검

## 11) 최종 원칙 요약
- “더 많은 정보”보다 “더 빠른 이해”가 우선
- 신규 API 없이도 파생 집계/강조만으로 상업용 체감 구현 가능
- 수정 범위는 **반드시** `NBA.html / NBA.css / NBA.js`와 4개 탭 내부로 한정
