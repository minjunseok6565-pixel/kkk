## 1) 결론 요약

현재 브랜치 기준으로 이슈 1은 **"핵심 엔진 신설은 완료"**, **"실제 운영 경로 연결은 미완"** 상태다.

- 완료:
  - `pick_distribution.py`에서 PMF/CDF/EV/분산/분위수/tail 지표를 산출하는 분포형 모델이 구현됨.
  - 보호/스왑에 대해 `pick_semantics`를 사용한 시나리오 계산 경로가 들어가 있음.
- 미완:
  - 실제 valuation 실행 경로(`service.py -> deal_evaluator.py -> market_pricing.py`)는 여전히 `PickExpectation.expected_pick_number` 단일값 중심.
  - `context_v2.py`가 분포를 만들지만 production valuation 경로에서 사용되지 않음.
- 결과적으로 현재 체감 동작은 **v1 단일 기대순번 체계가 계속 우세**.

## 2) 구현 상태 상세 점검

## 2-1. 분포형 픽 자산 엔진 존재 여부

### 확인 결과
`trades/valuation/pick_distribution.py`에 아래가 구현되어 있음.

- `PickDistributionBundle`:
  - `pmf`, `cdf`, `ev_pick`, `variance`, `p10/p50/p90`, `tail_upside_prob`, `tail_downside_prob`, `source_coverage`
- `build_pick_distributions_from_standings()`:
  - standings 기반 baseline PMF 생성
  - protection 검증/정규화(`normalize_protection`)
  - protection/swap semantics 적용(`resolve_pick_protection`, `resolve_swap_outcome`)
  - PMF 합 검증 및 정규화

### 판단
구현구상/상세설계에서 요구한 "단일 기대값 -> 분포형"의 계산 핵심은 **코드 레벨로 구현 완료**.

---

## 2-2. v2 오케스트레이션 연결 여부

### 확인 결과
`trades/valuation/context_v2.py`에서
- standings + lottery rules를 사용해 `build_pick_distributions_from_standings()` 호출,
- 결과를 `ValuationContextV2.pick_distributions`에 적재.

### 판단
v2 context 내부 연결은 되어 있음. 다만 이것만으로는 운영 경로 반영이 완료된 것이 아님.

---

## 2-3. 실제 valuation 경로 반영 여부(핵심)

### 확인 결과
운영 경로는 여전히 아래 흐름.

- `service.py`에서 `build_repo_valuation_data_context()` 호출
- 이 함수는 기본적으로 `build_pick_expectations_from_standings()`(선형 1..30)로 `PickExpectation` 생성
- `deal_evaluator.py`는 `provider.get_pick_expectation()`을 통해 기대순번을 읽어 `market_pricing.py`에 전달
- `market_pricing.py`의 `_price_pick()`은 `expected_pick_number` 단일값으로 `PICK_EXPECTED_NUM_BONUS`, `PICK_PROTECTION_EXPECTATION` 등을 계산

### 판단
이슈 1의 본 목표였던 "분포형 자산 기반 가격화"는 **운영 경로 미적용**.
현재는 `compat_expected_pick_number` 관점의 과도기조차 완전 연결되어 있지 않다.

## 3) 추가로 해야 할 작업 (권장 우선순위)

## P0 (반드시 선행)

1. **valuation 실행경로에 v2 pick distribution 주입 경로 추가**
   - 방법 A: `ValuationDataProvider`에 distribution accessor 추가
   - 방법 B: `deal_evaluator`가 `ValuationContextV2`를 직접 받아 분포 지표 전달
2. **`market_pricing._price_pick()` 분포 지표 소비 로직 추가**
   - 최소: `ev_pick + variance`
   - 권장: `tail_upside_prob/tail_downside_prob`, `p10/p50/p90` 기반 risk 조정
3. **v2 on 시 기존 근사 이중적용 차단**
   - `PICK_PROTECTION_EXPECTATION` / `SWAP_EXERCISE_PROB`의 중복 반영 방지 스위치 구현

## P1 (dual-read 안정화)

4. **dual-read telemetry를 실제 v1 비교값과 연결**
   - 현재 `context_v2.py`는 `collect_v1_v2_diff(v1_metrics={...})`에 실측 v1 지표를 넣지 않아 diff 의미가 약함.
5. **통합 테스트 추가**
   - 동일 입력에서 v1/v2 pick value 차이 검증
   - 보호/스왑이 있는 케이스에서 v2가 분산/꼬리 리스크를 반영하는지 검증

## P2 (튜닝/운영)

6. 시나리오 note/coverage를 decision explain에 연결
7. 시즌 룰 부재/standings 누락 시 fallback 정책을 운영 수준으로 고정
