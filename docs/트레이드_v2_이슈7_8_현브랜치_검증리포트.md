## 2) 결론 요약

### 이슈 7 (CAP_FLEX)
**판정: 부분 달성 (구성요소는 구현됨, 최종 스코어 경로는 미완 전환)**

- `contract_texture.py`가 `guaranteed_commitment`, `control_direction`, `trigger_risk`, `toxic_risk` 등 계약 질감 지표를 계산함.
- `package_effects.py`의 CAP_FLEX 경로는 기존 `salary*years` 직접 계산이 아니라 `build_contract_textures()` 결과를 받아 커미트먼트 델타를 계산함.
- `cap_ledger.py`가 `current_operating_room`, `apron_pressure`, `optionality_profile`, `posture_adjusted_flex`를 계산하도록 구현되어 있음.
- **하지만** `cap_ledger.score_cap_flex_delta()`는 실제 패키지 의사결정 점수 계산 경로에서 아직 호출되지 않음(컨텍스트/진단 집계 위주).

즉, “선형 근사 해소를 위한 데이터 모델”은 갖춰졌고 package_effects 일부 반영도 되었지만,
상세설계의 최종 목표( `_cap_flex_adjustment` 선형 커미트 경로 제거 + `score_cap_flex_delta(before, after)` 전환)는 완료되지 않았다.

### 이슈 8 (중복 평가)
**판정: 부분 달성 (단일 archetype 제거는 달성, 로테이션 맥락은 약함)**

- `package_effects.py`의 체감 감소 로직은 단일 archetype bucket 대신 `RoleTexture` 5축 벡터 유사도(`_texture_overlap`) 기반으로 동작함.
- `DIMINISHING_RETURNS` 스텝에 pairwise overlap matrix가 남아 설명 가능성도 개선됨.
- **하지만** 로테이션 맥락은 “초과 유입 인원 수(excess count)” 및 하위 가치 선수 waste 처리 수준이며, 실제 팀 로테이션/분 단위 맥락 입력은 직접 반영되지 않음.

즉, “단일 버킷 → 다차원 중복”은 반영됐으나,
요구 문구의 “+ 로테이션 맥락”은 아직 soft heuristic 수준이다.

## 3) 세부 검증 결과

## 3-1. 이슈 7 세부

### 확인된 구현

1. **Contract Texture 구현 완료**
   - `build_contract_textures()`에서 잔여 보장총액/옵션 방향/트리거 리스크/매칭 유틸/독성 리스크를 계산.

2. **Package CAP_FLEX 경로에 texture 주입**
   - `_cap_flex_adjustment()`가 선수별 계약 텍스처를 생성하고,
   - `guaranteed_commitment`에 옵션/트리거/독성 가중 multiplier를 적용해 델타를 계산.

3. **Cap Ledger 구현 완료(모듈 단위)**
   - 팀 단위로 cap room/apron/optionality/liquidity/posture 조정 유연성 점수 생성.

### 미완/추가 필요

1. **CapLedger 기반 before/after delta 미반영**
   - `score_cap_flex_delta(before, after)` 함수는 존재하지만 package_effects 경로에서 사용되지 않음.
   - 현재 CAP_FLEX 실점수는 여전히 `_cap_flex_adjustment()` 내부 계산식(스케일/캡) 중심.

2. **설계 문서의 legacy 교체 포인트 미완료**
   - 상세설계 문서 2.5-G 요구(선형 `delta_commitment` 경로 제거)가 아직 남아 있음.

## 3-2. 이슈 8 세부

### 확인된 구현

1. **단일 archetype 분류 제거/대체**
   - `RoleTexture` 5축(creation/spacing/rim_pressure/defense/connector) 기반 비교 도입.
   - `DIMINISHING_RETURNS`가 pairwise overlap 행렬을 통해 중복 페널티를 계산.

2. **테스트 커버 확인**
   - `test_diminishing_returns_uses_texture_overlap_matrix`에서 overlap matrix 메타 존재를 검증.

### 미완/추가 필요

1. **로테이션 맥락 반영 깊이 부족**
   - 현 로직은 `len(incoming)-len(outgoing)` 중심의 슬롯 초과 페널티로 구성되어,
     실제 전술 로테이션(핸들러 중복, 윙 분배, 빅맨 타입 조합)의 팀별 컨텍스트 반영이 제한적.

2. **Depth/Hole 보조 로직은 여전히 G/W/B 버킷 기반**
   - `classify_depth_buckets()`에 의존한 구조는 유지 중.
   - 이는 이슈8 본체(diminishing)와 별개이나, “다차원 중복 + 로테이션 맥락”을 끝까지 밀려면
     depth/hole 보조 계산도 texture 친화적으로 점진 이관 필요.

## 4) 현재 브랜치 기준 추가 권장 작업

우선순위 순 제안:

1. **CAP_FLEX 최종 전환(최우선)**
   - package 평가 시 team before/after `CapLedgerView`를 구성해
     `score_cap_flex_delta()` 결과를 CAP_FLEX 공식 점수로 승격.
   - `_cap_flex_adjustment`의 내부 선형 커미트 스케일 경로는 단계적 폐기(플래그 하 전환).

2. **로테이션 맥락 주입 강화**
   - `DecisionContext` 또는 기존 tactics/rotation snapshot에서 최소 입력(핵심 볼핸들러 수요, 빅-윙 배치 부담)을 읽어
     `ROSTER_EXCESS_WASTE` 가중치에 팀 맥락을 곱하는 방식부터 시작.

3. **Depth/Hole 보조 로직 v2화**
   - `classify_depth_buckets()` 단독 의존을 줄이고,
     `RoleTexture` 축 기반 공급 부족(예: creation deficit, rim_pressure deficit)을 병렬 산출.

4. **관측 지표 추가**
   - v1/v2 diff에 `rotation_conflict_delta`, `ledger_flex_delta`를 추가해
     튜닝 시 회귀를 더 빨리 감지.
